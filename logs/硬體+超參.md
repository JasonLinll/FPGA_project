# Transformer Attention + FFN Co-Sparse Accelerator — 機制與硬體規格 (v6)

**Target**: Xilinx KV260 FPGA (117K LUT, 1248 DSP, 4GB DDR4, ~26 Mb BRAM)
**Scope**: LLaMA-family decoder-only Transformer (TinyLlama, Qwen2.5, LLaMA-3.2, Mistral, ...)，GQA + SwiGLU + RoPE
**前提**: AWQ INT4 weight-only 量化模型 (premise, 非可選項; 硬體 datapath 依賴之, 不支援非 AWQ 路徑)

---

## 0. 系統總覽

兩個正交稀疏機制 + 全棧量化：

| 區塊 | 機制 | selection 粒度 | 省的資源 |
|------|------|---------------|---------|
| Attention (Q·K·V) | RAD-Cascade + KIVI-V | token-level (active dim + Global Top-N) | Q·K MAC + KV cache BW |
| FFN (SwiGLU) | CETT activation sparsity | neuron-level (真實貢獻 ‖n_i‖ 閾值) | down MAC + down 權重 BW |

兩機制統計獨立 (整合實測交互項 ≈ +0.09 pp，近線性疊加)，共用 Sparse Gather Engine。

## 0.1 全棧位宽與資料流 (已實測確認)

| 資料 | 位寬 | 說明 |
|------|------|------|
| 權重 (W_q/k/v/o, W_gate/up/down) | **INT4** (AWQ) + per-group FP16 scale (group 128) | DDR 存 INT4, register dequant 回 FP16 算 |
| 激活 hidden_states | **FP16** | AWQ weight-only, 激活保持高精度 |
| Q / K / V 向量 (投影輸出) | **FP16** → 進 attention PE 轉定點 (見下) | |
| Q·K 運算 | **INT8 × INT8** | Q/K 進 PE 轉 INT8 (對齊 SIGN sign-XOR + INT8 MAC) |
| K-anchor running-max | **FP16** 片上寄存器 | 只比大小選 anchor, 不參與 MAC, 0 DDR BW |
| KV cache: K | **INT8** | 1 byte/元素, 寫/讀 DDR |
| KV cache: V | **INT4** (KIVI group=32) + scale/zero | ~0.625 byte/元素 (含 group scale 開銷) |
| attn_w (softmax 輸出) | **FP16** | 與 V 解壓後 FP16 對齊 |
| attn_w × V 運算 | **FP16 × FP16** | V 從 INT4 解壓回 FP16 後算 (同權重處理) |

**關鍵: 各 linear (投影/o_proj/gate/up/down) 是 FP16 MAC (INT4 權重 register dequant)**。
- AWQ 省的是**權重 DDR BW** (INT4 vs FP16 ≈ 省 74%, 含 group scale)，**不省 linear MAC** (MAC 仍 FP16)。
- 省 MAC 來自 RAD attention (active dim + Top-N) 與 CETT FFN (down 稀疏)，非量化。

**權重 BW 精算**: INT4 + group=128 的 FP16 scale = (128×4 + 16) bit / 128 = 4.125 bit/weight → 省 74% (非 75%)。

## 1. 機制 (model-agnostic)

### Notation

| Symbol | 定義 |
|--------|------|
| `L` | 層數 |
| `H_Q` / `H_KV` | Query / KV head 數 (GQA) |
| `D` | head_dim |
| `P` | num_pairs = D / 2 |
| `S` | sequence length |
| `A` | active pairs per query |
| `n_K` | K-anchor pair 數 |
| `N` | Global Top-N candidate 數 |
| `G` | KIVI V group size |
| `d_ff` | FFN intermediate size |
| `τ_CETT` | FFN CETT 閾值 (PPL+1% 標準對應) |
| `ε_i` | 第 i 層 FFN neuron 貢獻閾值 (τ_CETT 反推, 離線校準) |

### Layer-wise dispatch

| Layer | Attention | FFN |
|-------|-----------|-----|
| Layer 0 | 異質: 跑滿 D dim (active=P, n_K=0), INT8 Q·K + INT4 V, 不套 RAD selection | CETT activation sparsity |
| Layer 1 ~ L-1 | RAD-Cascade (active + SIGN + Global Top-N) + KIVI-V | CETT activation sparsity |

Layer 0 異質原因: per-row attention 分散度高 (max ≈ 1/S), RAD selection 不適用; 跑滿維度, 後續層才省。

### 1.A Attention: RAD-Cascade

量化: Q, K per-token INT8 (進 PE 後轉)。

#### Stage 1: Active path + SIGN 補償 (Q·K 端)

**Active dim 選擇** (INT8 MAC, full-D-dim einsum with mask):
- 每 query 選 `A` 個 active pair，兩類組成：
  - `n_K` 個 **K-anchor pair**: per-position streaming running-max of |K| (FP16) 選 top-n_K (見 2.7)
  - `A - n_K` 個 **dynamic pair**: per-query |Q| pair-mag (INT8 L1) 選 top, anchor scatter 排除避免重複
- pair → dim: `full_dim_mask = pair_mask.repeat_interleave(2)`
- `active_score = einsum(Q_b ⊙ full_dim_mask, K_b)` (mask 乘法後全 D 維 einsum, 等價只算 active 維)
- 不使用 Q-anchor (n_Q = 0, 見 2.5)

**SIGN 補償** (純 sign, 1-bit sign(K) + 8-bit |Q| 累加, 零乘法器):
```
sign_k = (K_b >= 0) ? +1 : -1
comp   = einsum(Q_b ⊙ dropped_mask, sign_k)    # Σ_d∈dropped  q_d · sign(k_d)
stage1 = active_score + β × comp
```
- β = 1.0 (Theorem 1)
- **純 sign-only, 無 mean(|K_drop|) 縮放** (sign-XOR 直接進 adder tree, 不需 |K| 統計表)

**Theorem 1** (SIGN unbiased): 對稱分布下 E[sign(k)·k] = mean(|k|), β=1.0 為 unbiased estimator。

#### Stage 2: Global Top-N selection (soft-tail, 非硬截斷)

```
full_score    = einsum(Q_b, K_b)                       # 全 D 維 INT8 (候選用)
stage1_masked = where(valid_causal, stage1, -inf)
topN_idx      = topk(stage1_masked, N)                  # row-global top-N
cand_mask     = scatter(topN_idx) & valid_causal
final_score   = where(cand_mask, full_score, stage1)    # 候選用 full, 非候選保留 stage1 近似
```

- **核心 (soft-tail)**: Top-N 選「誰用精算 full_score」，**非候選 token 保留 stage1 近似** (不丟棄、不設 -inf)。尾巴 token 仍以近似權重參與 softmax。
- **soft-tail 為硬性要求 (實測證據)**: 嘗試 hard-truncation (非候選設 -inf, 只讀 top_n V) → prefill PG-19 PPL **+183% 崩潰** (每 query 表徵被砍到只剩 N 個 target, 累積毀掉所有位置)。soft-tail 保留尾部小權重才不崩。詳見 dead directions。
- `N = f(S)` (見 1.C)；row-global，非 per-block 配額。

#### Stage 3: KIVI-V + attn_w

- attn_w (softmax 輸出): **FP16** (不量化)
- V: per-token group-wise INT4 (KIVI, Liu et al. 2024), default `G = 32`
- **attn_w × V**: V 從 INT4 解壓回 FP16, FP16 MAC, **讀全部 V** (soft-tail 需全 V 參與)
```
For each group g of G channels:
    scale_g = (max-min)/15;  V_q = round((V-min)/scale_g) ∈ [0,15]
attn_out = attn_w(FP16) @ V_dequant(FP16)
```
- **V BW 省來自 INT4 壓縮 (~75%)，非 top_n 截斷** (硬截斷已否決, 見上)。

### 1.B FFN: CETT Activation Sparsity

#### 原理
SwiGLU: `out = W_down( SiLU(x·W_gate) ⊙ (x·W_up) )`。每 neuron 對 MLP 輸出的真實貢獻向量 `n_i = h_i · W_down[:,i]` (h_i = SiLU(gate_i)·up_i)。跳過低貢獻 neuron → 其 W_down[:,i] 不算/不載 → 省 down MAC + down 權重 BW。

#### Selection: CETT ε 閾值 (training-free, 動態)
判據用 neuron **真實貢獻範數** (含 down 投影, 比純 gate proxy 準):
```
‖n_i‖ ≈ |h_i| × ‖W_down[:,i]‖           # h_i 標量 × down 列範數 (SRAM 常數)
跳過 ‖n_i‖ < ε_i 的 neuron               # 動態, 砍量隨輸入/token 變
```

**CETT 度量** (TDA Eq.20, 校準 ε 用):
```
D = { i | ‖n_i‖ < ε }
CETT = ‖ Σ_{i∈D} n_i ‖₂ / ‖ MLP_output ‖₂     # 被砍 neuron 輸出「向量和」範數 / 完整輸出範數
```
- 分子為**向量和的範數** (有方向抵消)，非標量平方和。TinyLlama 上向量抵消可忽略 (與標量近似差 <2%)，但定義以 TDA Eq.20 為準。

#### 標準: PPL 漲 1% (文獻泛化, 非 overfit 固定比例)
- τ_CETT = 使 validation PPL 漲 ~1% 的 CETT 值 (Sparsing Law arXiv 2411.02335 證明此標準泛化性最強、高稀疏退化最小)。
- 每層 ε_i 由 τ_CETT 反推 (二分搜，離線校準，~8 chunk 收斂)。
- **τ_CETT 是 model-agnostic 標準** (「損失 1% PPL」任何模型同義); sparsity 是結果 (隨 model/輸入自適應)，不寫死。

#### prefill 與 generation 統一動態
- prefill 與 generation **都用 CETT ε 動態砍** (每 token 算 ‖n_i‖ 比 ε_i)，砍量隨輸入變。
- **不使用固定 mask 複用** (TDA-style prefill 定 mask、generation 重用): 實測 generation 固定 mask 與動態需求重合度僅 69.9% (淺層 L0=43.6%), 砍 43% → ROUGE 崩 5.26; 換 Llama-3.2-3B 重合度 68.6% (非規模問題, 固定 mask 本質不適用)。詳見 dead directions。

#### 機制邊界 (為何只省 down)
training-free + 不改激活函數 + SwiGLU 三約束下，**只能省 down**:
- 要選哪些 neuron 砍，須先算 h_i = SiLU(gate)·up → **gate/up 必須全算** (FP16 MAC, 不省)。
- SwiGLU (SiLU) 無硬零 (不像 ReLU)，激活不可廉價預測 → 無法免訓練 predictor 預測 mask 來省 gate/up。
- 文獻中省 gate/up 者皆需 (a) train predictor (DejaVu/PowerInfer) 或 (b) 改激活函數為 ReLU²/FATReLU 並續訓 (ProSparse 80%) → 違反 training-free + no-modification。
- TDA 論文亦明言「only down projection reduced <50%, L2 norm 仍需算」→ 只省 down 是約束的數學本質，非實作不足。

#### CETT vs 舊 gate-proxy (為何升級)
- gate proxy (|SiLU(gate)|×‖W_up‖) 上限 ~20% (gate⊥up 統計獨立, corr≈0)，砍 25% 即 ROUGE 崩。
- CETT (含 down 投影 ‖n_i‖) + ε 動態 + PPL+1% 標準 → 無損砍 ~43% (TinyLlama PG-19 PPL +0.94%, XSum ROUGE-1 drop 0.70)，**翻倍且更嚴謹**。

### 1.C 序列級自適應 (active 與 top_n 隨 S 配置)

attention 兩稀疏參數依**序列長度 S** 在 prefill (S 已知) 配置一次，**整段固定** (非 per-query → MAC 陣列配置一次, 無 pipeline 氣泡)。roofline: 短序列 compute-bound (省 MAC), 長序列 memory-bound (精度優先, BW 靠 top_n + KV 量化省)。

**active=f(S)** (雙配置, 比例 per-model 校準, 須 DMA 對齊):
| S | active ratio | 場景 |
|---|--------------|------|
| 短序列 (S < S_cut) | 低 (~25%) | MAC 省優先 |
| 長序列 (S ≥ S_cut) | 高 (~37.5%) | 精度優先 |

- active dim **必須 DMA 對齊** (匯流排寬度倍數)；非對齊值 (如 20 dim=160-bit) 排除。
- 實際 dim 與切點 S_cut 為 per-model calibration (PPL 轉折定)。

**top_n=f(S)** (三檔):
| S | top_n |
|---|-------|
| S < 128 | 8 |
| 128 ≤ S < 256 | 16 |
| S ≥ 256 | 32 |

- top_n 影響 Stage 2 gather 候選數 (gather 層面可變, 不破壞 MAC 陣列)。短序列 Stage 2 占 MAC 比例高 (省 top_n 省大量 MAC); 長序列占比小, top_n=32 維持精度。

**為何序列級固定而非 per-query / per-layer**:
- per-query 動態 active → MAC 陣列每 query 重配 → 氣泡。序列級配置一次, 無氣泡。
- per-layer 差異化 active 無 model-agnostic 免校準判據, 不採用。
- top_n 動態為 gather 層面, 不觸 MAC 陣列, 故隨 S 配置無氣泡。

### 完整 dataflow
```
Per layer:
  [Attention]
    QKV 投影: hidden(FP16) @ W_qkv(INT4 dequant FP16) → Q/K/V (FP16)
    RoPE (FP16) → Q,K
    K: FP16 |·| 更新 running-max 錨點 (片上寄存器); 量化 INT8 寫 cache
    V: 量化 INT4 寫 cache; Q: 轉 INT8 留片上
    Per query:
        1. K-anchor: top-n_K of 當下 K_runmax (FP16)
        2. Dynamic : top-(A-n_K) of |Q| pair-mag (INT8, anchor 排除)
        3. Stage 1 : active_score (INT8 mask einsum) + β·SIGN(q·sign k, 零乘法)
        4. Stage 2 : full_score(INT8) on top-N candidate; final = where(cand, full, stage1)  [soft-tail]
        5. Softmax → attn_w (FP16)
        6. KIVI INT4 V dequant FP16 → attn_w @ V (FP16, 讀全部 V) → attn_out
    o_proj: attn_out(FP16) @ W_o(INT4 dequant FP16)
  [FFN]
        7. gate = x·W_gate, up = x·W_up   (INT4 dequant FP16 MAC, 全算)
        8. h_i = SiLU(gate_i)·up_i
        9. ‖n_i‖ = |h_i| × ‖W_down[:,i]‖(SRAM); keep = ‖n_i‖ ≥ ε_i  (動態)
       10. down: gather keep 的 W_down → attn_w(keep) → ffn_out  (省 down MAC + BW)
```

## 2. 軟體超參如何測得

### 2.1 Metric Hierarchy
| Metric | 用途 | 可信度 |
|--------|------|-------|
| KL max | n_K sweep | 跨 model 驗證可信, primary |
| CETT (PPL+1%) | FFN ε 校準 | 文獻泛化標準 |
| End-to-end (HellaSwag/PG-19/XSum) | ground truth | 任何機制變動必須驗 |

### 2.2 β (SIGN) = 1.0 (Theorem 1 + sweep)
### 2.3 KIVI group G = 32 (KIVI default)
### 2.4 active_dim ratio A/P — 序列級雙配置 (見 1.C)
- 預設 0.375 (24 dim); 短序列 0.25 (16 dim); DMA 對齊 (16/24/32 dim)
- 新 model: 三 candidate (25/37.5/50%) + end-to-end 實測
### 2.5 不使用 Q-anchor: n_Q=0 (TinyLlama 1Q vs 0Q gap 0.01pp; SparQ cross-model 皆無; register 保留 fallback)
### 2.6 K-anchor 數量 n_K ∈ {1,2,3,4} per-model (KL max → end-to-end)
### 2.7 K-anchor 選法: per-position streaming running-max
```
Init K_runmax[L,kv_h,p]=0
Per new K_i: K_runmax[p] ← max(K_runmax[p], max(|K_i[2p]|,|K_i[2p+1]|))   # FP16 片上
Per query:   K_anchor = top-n_K of 當下 K_runmax
```
驗證 (TinyLlama PG-19, 三估計法): K_est.mean +1.70% / running-max 最終 +1.68% / per-position +1.63% (差<0.1pp)。採 per-position (硬體最簡: P 維 FP16 running-max 寄存器, 零 offline 零額外存儲)。long context 自動抓 attention sink。
### 2.8 Global Top-N 數量 N=f(S) (序列級, 見 1.C)
### 2.9 FFN: CETT 閾值 τ_CETT (PPL+1% 標準)
```
給 τ_CETT (PPL+1% 對應, TinyLlama ≈ CETT 0.08):
  離線: 每層二分搜 ε_i 使該層 CETT ≈ τ_CETT (~8 chunk 收斂)
  runtime: 砍 ‖n_i‖ < ε_i 的 neuron (動態)
```
- TinyLlama: τ_CETT=0.08 → 平均 sparsity ~43% (PG-19 PPL +0.94%); per-model 校準, 不寫死跨 model。
- ε_i 跨分布穩定 (PG-19 vs XSum 校準一致), 收斂快 (N=10 即穩) → 離線校準可行。

## 3. Calibration / Deployment Flow

### 3.1 Pre-deployment (offline, one-time)
1. Load AWQ model + tokenizer
2. Attention active ratio (25/37.5/50%) sweep → KL max + sanity → top-2
3. n_K sweep (KL max {1,2,3,4}) → top-2
4. FFN: 算每層 ‖W_down[:,i]‖ 範數 (SRAM 表); 二分搜 ε_i 使 CETT=τ_CETT (PPL+1%)
5. Final: full HellaSwag (attn) + PG-19 PPL (整合) + XSum ROUGE (generation)
6. Write registers + load ‖W_down‖ 範數向量

### 3.2 Runtime (no calibration set)
- K-anchor: per-position streaming running-max (FP16 片上)
- FFN: gate/up 算完 → ‖n_i‖ 比 ε_i (動態砍 down)
- n_K / N / ε_i / G: 固定

### 4.3 BRAM / SRAM tables
| Table | 位置 | Size | Update |
|-------|------|------|--------|
| K_runmax | BRAM | L×HKV×P_MAX×16 bit (FP16) ≈ 1.4 KB | per-token max-compare (streaming) |
| W_down_norm + ε_i | DDR→SRAM (逐層載入) | 單層 d_ff×8 bit (TinyLlama 5.6 KB) + L×ε | offline 算好 |

**W_down_norm 不常駐 BRAM**: 逐層執行, 第 i 層只需該層 INT8 範數 (5.6 KB), DDR 載入 + Ping-Pong 預取下一層。單層 5.6 KB vs 單層 FFN 權重 ~17-35 MB → BW 佔比 ~0.02%, 可忽略。

### 4.4 整體架構
```
KV260 FPGA
├── QK-PU (Q·K, scalar-reduction dataflow)
│   ├── PE Lane × LANE_COUNT: INT8 MAC + Adder Tree (active + SIGN 共用樹)
│   ├── Cross-bar dim selector (pair-mask → active dim)
│   ├── Q/K FP16→INT8 量化單元
│   └── K-runmax update unit (per-token pair-abs-max + FP16 running-max)
├── Unified Top-K Engine (≤32)          ── K-anchor / Dynamic / Global Top-N 分時共用
├── Softmax (base-2 + reciprocal LUT) → attn_w FP16
├── V-PU (attn_w·V, vector-accumulate dataflow)
│   └── 16×16 systolic (INT4 V dequant→FP16 × FP16 attn_w, KIVI per-group scale)
├── Sparse Gather Engine                ── attn Stage 2 token / FFN down neuron 共用 DMA
├── CETT Filter Unit                    ── FFN ‖n_i‖ vs ε_i 動態裁決
├── Weight Dequant Unit                 ── INT4 → FP16 (per-group scale), 供所有 linear
└── Memory (K/V/Q Buffer + K_runmax [BRAM]; W_down_norm + AWQ weights [DDR4])
```

### 4.5 硬體資源共用

**① 統一 Top-K Engine (≤32) — 三 stage 共用**
K-anchor (k≤4)、Dynamic pair (k≈A-n_K)、Global Top-N (k=N≤32) 皆「分數流選 top-k」，k≤32 且時間不重疊。共用 32-entry sorted shift register，k 由 register 配置。省 2 套選擇器。

**② FFN selection 不併入 Top-K Engine — 獨立 CETT Filter**
FFN 留 neuron 數巨大 (千級)，streaming 全排序面積爆炸。改用 **CETT 閾值過濾** (‖n_i‖ vs ε_i, 過線即留)，非全排序。獨立成 CETT Filter Unit。

**③ Sparse Gather Engine — attn Stage 2 與 FFN down 共用**
Stage 2 token gather 與 FFN down neuron 權重 gather，「地址生成 → read request → 收亂序回傳」狀態機相同。共用同一 DMA gather 控制器。解決 memory wall 核心整合。

**④ Adder Tree — Active MAC 與 SIGN 共用**
SIGN = Σ|Q_d|·sign(K_d): sign(K) 對 Q 做 XOR 取反 (不需乘法器), 進 active path 同一 Adder Tree。active 與 SIGN time-multiplex 共用。

**⑤ Weight Dequant Unit — 所有 linear 共用**
INT4 權重 + per-group FP16 scale → FP16, 供投影/o_proj/gate/up/down。register-level (送進 MAC 前一刻 dequant), 不增 DDR BW。

**不共用: QK-PU 與 V-PU MAC** — reduction 方向正交 (QK 歸約成 scalar 用 adder tree; attn_w·V 跨 token 累加成 vector 用 vector accumulator)，硬塞 Mux 布線面積 > 省的乘法器。保持兩套獨立 datapath。

### 4.5.1 序列級自適應配置單元 (active/top_n = f(S))
```
At prefill (S known):
    active_pairs = (S < S_cut) ? A_short : A_long   // 配置 cross-bar 一次
    top_n        = (S<128)?8 : (S<256)?16 : 32      // Unified Top-K Engine 的 k
```
純比較器, ~0 額外 DSP, 序列開始配置一次 (無氣泡)。DMA 對齊保證。

### 4.6 K-runmax update unit (per-position streaming, FP16)
```
Per K_i vector:
    Step1 pair abs-max:  K_pair[p] = max(|K_i[2p]|,|K_i[2p+1]|)   // P 並行 FP16 compare
    Step2 running-max :  K_runmax[p] ← max(K_runmax[p], K_pair[p])
Per query: 送當下 K_runmax 給 Unified Top-K Engine 選 top-n_K
```
~0.5K LUT, 0 DSP, 1 cycle/K vector。FP16 (只比大小, 不需定點對齊)。

### 4.7 Unified Top-K Engine (≤32)
```
32-entry sorted register (score 16-bit + index), k 由 reg 配置:
  每新分數 s: if s > min_entry: 插入冒泡淘汰最小   // O(k) compare, 1 score/cycle
三 stage 分時: K-anchor (k=n_K) / Dynamic (k=A-n_K) / Global Top-N (k=N, 輸出 index → Gather)
```
~2K LUT, 0 DSP。

### 4.8 Sparse Gather Engine (attn + FFN 共用)
```
輸入: index list (runtime)
狀態機: 地址生成 → read request → 收亂序回傳 → 對齊送 PE
  attn Stage 2: index = Global Top-N token → gather K row (補算 full_score)
  FFN down    : index = keep neuron → gather W_down[:,keep]
```
~1.5K LUT, 0 DSP (分時)。

### 4.9 CETT Filter Unit
```
‖n_i‖ = |SiLU(INT8 gate_i)·up_i| × W_down_norm[i]   // SiLU LUT × DDR 載入單層 INT8 範數
keep = ‖n_i‖ ≥ ε_i   (ε_i per-layer 常數, SRAM)      // 動態, 非全排序
跳過 neuron → 其 W_down[:,i] 不從 DDR 載入, down MAC 不算
```
~0.6K LUT, 0 DSP。

### 4.10 PE Lane / Cross-bar / Softmax / KIVI / Dequant
- **PE Lane** (QK-PU): active INT8 MAC + Adder Tree; SIGN sign-XOR(Q) 進同一 Adder Tree; time-multiplex
- **Cross-bar**: LANE×PE cascading mux, pair-mask → active dim, single-cycle
- **Softmax**: base-2 + reciprocal LUT; 輸出 attn_w FP16
- **V-PU (KIVI)**: 16×16 systolic, INT4 V per-group dequant→FP16, × FP16 attn_w
- **Weight Dequant**: INT4 + per-group FP16 scale → FP16, 所有 linear 共用

### 4.11 Layer 0 異質 mapping
物理 PE = LANE×PE = 48。Layer 0 跑滿 D: ceil(D/48) iteration。throughput 慢 2-3x, only 1/L layers, 整體 <10%。

## 5. 各機制節省 (待 characterization 精算)

分 prefill / generation × 序列長度 (memory-bound: BW 省為重):
- **權重 BW**: AWQ INT4 省 ~74% (所有 linear, 解析)
- **Q·K MAC**: RAD active+top_n 省 f(S) (S=512 ~55%, S=2048 待算; 解析)
- **KV cache BW**: K INT8 (省 50%) + V INT4 (省 75%); generation 每步讀全 cache 為大頭
- **FFN MAC**: CETT 砍 down, sparsity ~43% → down 省 43% → 全 FFN 省 ~14% (gate/up 全算)
- **FFN 權重 BW**: down 砍 neuron 不載 ~43% × down 部分

→ 完整 (階段 × S × 機制) MAC/BW 絕對值 (Bytes/MACs) + 相對% 表待 characterization 腳本精算。
