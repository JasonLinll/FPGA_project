# Transformer Attention + FFN Co-Sparse Accelerator — 機制與硬體規格 (v5)

**Target**: Xilinx KV260 FPGA (117K LUT, 1248 DSP, 4GB DDR4, 640 KB BRAM)
**Scope**: LLaMA-family decoder-only Transformer (TinyLlama, Qwen, LLaMA2/3, Mistral, ...)

---

## 0. 系統總覽

兩個正交稀疏機制：

| 區塊 | 機制 | selection | 省的資源 |
|------|------|-----------|---------|
| Attention (Q·K·V) | RAD-Cascade + KIVI-V | token-level (active dim + Global Top-N) | KV cache BW + MAC |
| FFN (SwiGLU) | Activation sparsity | neuron-level (gate proxy) | FFN 權重 BW (W_up/W_down) |

兩機制統計獨立 (整合實測交互項 ≈ +0.09 pp，近線性疊加)，共用 Sparse Gather Engine。

## 1. 機制 (model-agnostic)

### Notation

| Symbol | 定義 |
|--------|------|
| `L` | 層數 |
| `H_Q` / `H_KV` | Query / KV head 數 (GQA) |
| `D` | head_dim |
| `P` | num_pairs = D / 2 |
| `S` | sequence length |
| `A` | active pairs per query |
| `n_K` | K-anchor pair 數 |
| `N` | Global Top-N candidate 數 |
| `G` | KIVI V group size |
| `d_ff` | FFN intermediate size |
| `K_ffn` | FFN neuron 剪枝率 |

### Layer-wise dispatch

| Layer | Attention | FFN |
|-------|-----------|-----|
| Layer 0 | 異質: 跑滿 D dim (active=P, n_K=0), 8-bit Q·K + 8-bit V, 不套 RAD | activation sparsity |
| Layer 1 ~ L-1 | RAD-Cascade (active + SIGN + Global Top-N) + KIVI-V | activation sparsity |

Layer 0 異質原因: per-row attention 分散度高 (max ≈ 1/S), RAD selection 不適用; 跑滿維度, 後續層才省。

### 1.A Attention: RAD-Cascade

量化: Q, K per-token INT8。

#### Stage 1: Active path + SIGN 補償 (Q·K 端)

**Active dim 選擇** (8-bit MAC, full-64-dim einsum with mask):
- 每 query 選 `A` 個 active pair (default `A = round(0.375 × P)`)，兩類組成：
  - `n_K` 個 **K-anchor pair**: per-position streaming running-max of |K| 選 top-n_K (見 2.7)
  - `A - n_K` 個 **dynamic pair**: per-query |Q| pair-mag (INT8 L1) 選 top, anchor scatter 排除避免重複
- pair → dim: `full_dim_mask = pair_mask.repeat_interleave(2)` (consecutive pair (2j,2j+1))
- `active_score = einsum(Q_b ⊙ full_dim_mask, K_b)` (mask 乘法後全 64 維 einsum, 等價只算 active 維但避免手動 index)
- 不使用 Q-anchor (n_Q = 0, 見 2.5)

**SIGN 補償** (1-bit sign + 8-bit |Q| 累加):
```
sign_k = (K_b >= 0) ? +1 : -1
comp   = einsum(Q_b ⊙ dropped_mask, sign_k)    # Σ_d∈dropped  q_d · sign(k_d)
stage1 = active_score + β × comp
```
- β = 1.0 (Theorem 1)
- **無 mean(|K_drop|) 縮放**: 補償為純 Σ q·sign(k) (sign-only, 不需 |K| 統計表)

**Theorem 1** (SIGN unbiased): 對稱分布下 E[sign(k)·k] = mean(|k|), β=1.0 為 unbiased estimator。

#### Stage 2: Global Top-N selection

```
full_score    = einsum(Q_b, K_b)                       # 全 64 維 INT8 (候選用)
stage1_masked = where(valid_causal, stage1, -inf)
topN_idx      = topk(stage1_masked, N)                  # row-global top-N
cand_mask     = scatter(topN_idx) & valid_causal
final_score   = where(cand_mask, full_score, stage1)    # 候選用 full, 非候選保留 stage1 近似
```

- **核心**: Top-N 選「誰用精算 full_score」, **非候選 token 保留 stage1 近似** (不丟棄/不 mask -inf)。尾巴 token 仍以近似權重參與 softmax, 優於 hard-truncation。
- `N = 32` default (row-global, 非 per-block 配額)
- Stage 2 對 N 個 candidate 補算 full-64-dim Q·K (8-bit), 需從 K buffer 隨機 gather N 個 token

#### Stage 3: KIVI-V + P matrix 量化

- P matrix: per-row INT8 量化
- V: per-token group-wise INT4 (KIVI, Liu et al. 2024), default `G = 32`
```
For each group g of G channels:
    scale_g = (max-min)/15;  V_q = round((V-min)/scale_g) ∈ [0,15]
output = P_int8 @ V_dequant
```

### 1.B FFN: Activation Sparsity

#### 原理
SwiGLU: `out = W_down( SiLU(x·W_gate) ⊙ (x·W_up) )`。gate/up 雙重過濾使 `h_i = SiLU(gate_i)·up_i` 大量趨近 0。跳過低貢獻 neuron → 其 W_up[:,i]/W_down[i,:] 不載入 → 省 FFN 權重 BW。

#### Selection: gate proxy (training-free, no-up-load)
```
proxy[i] = | SiLU(INT8 gate_i) | × ||W_up[:,i]||
跳過 proxy 最低的 K_ffn 比例 neuron
```
- `||W_up[:,i]||`: 離線 L2 範數, per-model 常數, SRAM 駐留
- gate 本就要算 (SwiGLU 必要), 量化後 selection 不增 BW

#### 機制邊界 (設計依據)
廉價代理上限由 **gate 與 up 統計獨立性** 決定: 實測 `corr(|SiLU(gate)|,|up|) ≈ 0` (跨層)。真實 top neuron 約 28% 為 u 主導 (gate 看不出), 結構性損失下界。
- 砍 ~20% (retained ≥ 0.95): 接近完美選擇上界 (差 < 5 pp), PPL 損失極小
- 砍 > 30%: 需載動態 up 才選得準, 但載 up 抵消節省
- 結論: 不載動態 up 上限 ≈ 20-30% (物理邊界)

不採用 (已驗劣): `×||W_down||` (噪聲); 全精度 gate 破 50% (假省); 靜態永久死 neuron (SwiGLU 無, 死活動態); per-layer 差異化 (各層 retained 可砍率收斂)。


### 1.C 序列級自適應 (active 與 top_n 隨 S 配置)

attention 的兩個稀疏度參數依**序列長度 S** 在序列開始 (prefill, S 已知) 配置一次, **整段序列固定** (非 per-query → MAC 陣列配置一次, 無 pipeline 氣泡)。對應 roofline: 短序列 compute-bound (省 MAC), 長序列 memory-bound (精度優先, BW 靠 top_n 省)。

**active=f(S)** (雙配置, 比例隨 model 校準, 須 DMA 對齊):
| S | active ratio | 場景 |
|---|--------------|------|
| 短序列 (S < S_cut) | 低 (~25%) | MAC 省優先 (compute-bound) |
| 長序列 (S ≥ S_cut) | 高 (~37.5%) | 精度優先 (memory-bound) |

- active dim **必須 DMA 對齊** (匯流排寬度倍數)。非對齊值 (如 D=64 下的 20 dim=160-bit) 排除: 需兩次讀取 + shift 拼湊, 省 MAC 卻燒記憶體控制器面積/功耗
- 兩檔的實際 dim 與切點 S_cut 為 **per-model calibration** (見各 model config); 切點由多序列長度 sweep 之 PPL 轉折定, 非理論常數

**top_n=f(S)** (三檔):
| S | top_n |
|---|-------|
| S < 128 | 8 |
| 128 ≤ S < 256 | 16 |
| S ≥ 256 | 32 |

- top_n 影響 Stage 2 補算 (gather 候選數), gather 層面可變, 不破壞 MAC 陣列 (無氣泡)
- 短序列 Stage 2 占 MAC 比例高 (S=76 時 ~41%), 砍 top_n 省大量 MAC; 長序列 Stage 2 占比小 (~2.5%), top_n=32 維持精度

**為何序列級固定而非 per-query / per-layer**:
- per-query 動態 active → MAC 陣列每 query 重配 → pipeline 氣泡 (省的 MAC 被空轉/stall 吃回)。序列級整段同 active, 配置一次, 無氣泡。
- per-layer 差異化 active 無 model-agnostic 免校準判據 (effective rank 僅表線性可壓縮性, 非「可砍性」; attention 熵屬 token 維度而非 feature 維度), 故不採用。
- top_n 動態為 gather 層面 (補算候選數), 不觸及 MAC 陣列, 故可隨 S 配置而無氣泡。

### 完整 dataflow
```
Per layer:
  [Attention]
    Per new K_i (streaming):
        K_pair[p] = max(|K_i[2p]|, |K_i[2p+1]|)
        K_runmax[p] ← max(K_runmax[p], K_pair[p])           # per-position running-max
    Per query:
        1. K-anchor: top-n_K of 當下 K_runmax
        2. Dynamic : top-(A-n_K) of |Q| pair-mag (anchor 排除)
        3. Stage 1 : active_score (mask einsum) + β·SIGN(q·sign k)
        4. Stage 2 : full_score on top-N candidate; final = where(cand, full, stage1)
        5. Softmax → P; P 量化 INT8
        6. KIVI dequant V → P @ V → attn_out
  [FFN]
        7. gate = x·W_gate                                   # dense, 必算
        8. proxy = |SiLU(INT8 gate)| × ||W_up||(SRAM)
        9. keep = top-(1-K_ffn) by proxy
       10. gather keep 的 W_up/W_down → h → ffn_out           # sparse gather
```

## 2. 軟體超參如何測得

### 2.1 Metric Hierarchy
| Metric | 用途 | 可信度 |
|--------|------|-------|
| Spearman ρ | 僅 active dim ratio sweep | 機制存廢預測失效, 不可作機制決策 |
| KL max | n_K sweep | 跨 model 驗證可信, primary |
| Retained magnitude | FFN 剪枝率 | 直接對應 PPL |
| End-to-end (HellaSwag/PG-19) | ground truth | 任何機制變動必須驗 |

**「最優」紀律**: 任何「最優」宣稱須附對照上界 (FFN perfect oracle、anchor 三版對照), 不可只憑「測過的最好」。

### 2.2 β (SIGN) = 1.0 (Theorem 1 + sweep 驗證)
### 2.3 KIVI group G = 32 (KIVI default, 引用不重 sweep)
### 2.4 active_dim ratio A/P — 序列級雙配置 (見 1.C)
- 預設 default 0.375 (24 dim); 短序列 (S<256) 用 0.25 (16 dim)
- 落在 SparQ r/d_h ∈ {25%,50%} 範圍; active 候選須 DMA 對齊 (16/24/32 dim)
- 新 model: 三 candidate (25/37.5/50%) + end-to-end, 無單一事前指標, 必須實測
### 2.5 不使用 Q-anchor: n_Q=0 (TinyLlama 1Q vs 0Q gap 0.01pp; Qwen KL identical; SparQ cross-model 皆無; register 保留作 fallback)
### 2.6 K-anchor 數量 n_K ∈ {1,2,3,4} per-model (n_K/P ≈ 3-5%; KL max → end-to-end)

### 2.7 K-anchor 選法: per-position streaming running-max
每 query 用「截至當前位置累積 running-max」選 top-n_K。no offline calibration。
```
Init K_runmax[L,kv_h,p]=0
Per new K_i: K_runmax[p] ← max(K_runmax[p], max(|K_i[2p]|,|K_i[2p+1]|))
Per query:   K_anchor = top-n_K of 當下 K_runmax
```
**驗證** (TinyLlama PG-19, 三種 K-anchor 估計法對照): K_est.mean +1.70% / running-max 最終值 +1.68% / **per-position 當下 running-max +1.63%**。三者本質相同 (差<0.1pp), 採 per-position (硬體最簡: P 維 running-max 暫存器, 零 offline 零額外存儲)。
**long context 優勢**: running-max 自動抓 attention sink。

### 2.8 Global Top-N 數量 N=f(S) (序列級, 見 1.C)
- 三檔: S<128→8, 128≤S<256→16, S≥256→32 (default)
- row-global (非 per-block); 短序列小 top_n 省 Stage 2 MAC, 長序列 32 維持精度
### 2.9 FFN 剪枝率 K_ffn: per-model, default 對應 retained 門檻 τ=0.95 (TinyLlama ≈ 20%)
```
給 τ (default 0.95):
  For each layer: 用 proxy 算「砍到該層平均 retained ≥ τ 的最大剪枝率」→ per-layer K_ffn
```
- τ 單一可解釋超參; 各層 K_ffn 收斂 (差<5%) 可用全域近似; 上限見 1.B

## 3. Calibration / Deployment Flow

### 3.1 Pre-deployment (offline, one-time, ~30-40 min)
1. Load model + tokenizer
2. Attention active ratio (25/37.5/50%) sweep → KL max + sanity → top-2
3. n_K sweep (KL max {1,2,3,4}) → top-2
4. FFN: 算每層 ||W_up|| 範數; τ=0.95 推 per-layer K_ffn
5. Final: full HellaSwag (attn) + retained sanity (FFN)
6. Long context sanity: PG-19 window=2048
7. Write registers + load ||W_up|| 範數向量

### 3.2 Runtime (no calibration set)
- K-anchor: per-position streaming running-max
- FFN proxy: gate 算完用 SRAM 範數裁決
- n_K / N / K_ffn / G: 固定

### 4.2 Runtime 配置 register
```

reg_num_layers        [5:0]
reg_num_q_heads       [5:0]
reg_num_kv_heads      [5:0]

```

### 4.3 BRAM / SRAM tables
| Table | 位置 | Size | Update |
|-------|------|------|--------|
| K_runmax | BRAM | L×HKV×P_MAX×8 bit ≈ 0.7 KB | per-token max-compare (streaming) |
| W_up_norm | DDR (逐層載入) | 單層 d_ff×8 bit (TinyLlama 5.6 KB) | offline 算好, 不更新 |

**W_up_norm 不常駐 BRAM**: LLM 逐層執行, 第 i 層 FFN 只需該層 INT8 範數 (單層 5.6 KB)。放 DDR、FFN 階段逐層載入、Ping-Pong buffer 預取下一層藏延遲。
- 單層 5.6 KB vs 單層 FFN 權重 ~17-35 MB → BW 佔比 ~0.02-0.03%, 可忽略
- INT8 範數對 neuron 排序精度足夠 (範數僅決定相對大小)
- K/V/Q buffer 維持 ~448 KB 常駐

### 4.4 整體架構
```
KV260 FPGA
├── QK-PU (Q·K, scalar-reduction dataflow)
│   ├── PE Lane × LANE_COUNT: 8-bit MAC + Adder Tree (active + SIGN 共用樹)
│   ├── Cross-bar dim selector (pair-mask → active dim)
│   └── K-runmax update unit (per-token pair-abs-max + running-max)
├── Unified Top-K Engine (≤32)          ── K-anchor / Dynamic / Global Top-N 分時共用
├── Softmax (base-2 + reciprocal LUT) + P INT8 量化
├── V-PU (P·V, vector-accumulate dataflow)
│   └── 16×16 systolic (INT4 V × INT8 P, KIVI per-group scale extract)
├── Sparse Gather Engine                ── attn Stage 2 token / FFN neuron 共用 DMA
├── FFN Threshold Filter Unit           ── FFN proxy (動態閾值, 非全排序)
└── Memory (K/V/Q Buffer + K_runmax [BRAM]; W_up_norm + weights [DDR4])
```

### 4.5 硬體資源共用 (核心整合)

四個共用/不共用決策，每個附理由：

**① 統一 Top-K Engine (≤32) — 三 stage 共用**
K-anchor (k≤4)、Dynamic pair (k≈11)、Global Top-N (k=32) 皆為「分數流選 top-k」，k 全 ≤32 且時間不重疊 (anchor/dynamic 在 Stage1 前、Global Top-N 在 Stage1 後)。共用一個 32-entry sorted shift register / systolic 比較陣列，k 由 register 配置。**省 2 套選擇器。**

**② FFN selection 不併入 Top-K Engine — 獨立 Threshold Filter**
FFN k 巨大 (TinyLlama 留 ~4500 / 砍 ~1126)，streaming 維持千級排序網路面積爆炸。FFN 改用**動態閾值過濾** (算前幾 token 定及格線, 過線即留)，非嚴格全排序。獨立成 FFN Threshold Filter Unit。

**③ Sparse Gather Engine — attn Stage 2 與 FFN 共用**
Stage 2 token gather 與 FFN neuron 權重 gather，雖資料寬度不同，但「地址生成 → 發 read request → 收亂序回傳」狀態機完全相同。共用同一 DMA gather 控制器。**解決 memory wall 的核心整合。**

**④ Adder Tree — Active MAC 與 SIGN 共用**
SIGN = `Σ |Q_d|·sign(K_d)`：用 sign(K) 對 Q 做 XOR 取反 (不需乘法器)，再進 active path 的同一 Adder Tree。active 與 SIGN time-multiplex 共用整棵加法樹。

**不共用: QK-PU 與 V-PU 的 MAC** — reduction 方向正交 (QK 歸約成 scalar 用 adder tree; PV 跨 token 累加成 vector 用 vector accumulator)，硬塞所需 Mux 布線面積 (FPGA 布線資源緊張) > 省下的乘法器。保持兩套獨立 datapath。

### 4.5.1 序列級自適應配置單元 (active/top_n = f(S))
prefill 時 S 已知, 序列開始用比較器選 active 與 top_n 檔位, **整段固定**:
```
At prefill (S known):
    active_pairs = (S < 256) ? 8 : 12         // 16dim : 24dim, 配置 cross-bar 一次
    top_n_base   = (S<128)?8 : (S<256)?16 : 32 // Unified Top-K Engine 的 k
```
- 純比較器, ~0 額外 DSP, 序列開始配置一次 (非 per-query, 無氣泡)
- active 切點配置 cross-bar dim selector 寬度; top_n 配置 Top-K Engine 的 k
- DMA 對齊保證 (16/24 dim = 128/192-bit)

### 4.6 K-runmax update unit (per-position streaming)
純 streaming running-max, 無需存歷史序列:
```
Per K_i vector [D bit]:
    Step1 pair abs-max:  K_pair[p] = max(|K_i[2p]|,|K_i[2p+1]|)   // P 並行 8-bit compare
    Step2 running-max :  K_runmax[p] ← max(K_runmax[p], K_pair[p])
Per query: 送當下 K_runmax 給 Unified Top-K Engine 選 top-n_K
```
~0.5K LUT, 0 DSP, 1 cycle/K vector。

### 4.7 Unified Top-K Engine (≤32)
```
32-entry sorted register (score 16-bit + index), k 由 reg 配置:
  每個新分數 s: if s > min_entry: 插入冒泡淘汰最小   // O(k) compare, pipeline 1 score/cycle
三 stage 分時:
  K-anchor    : 輸入 K_runmax, k=n_K
  Dynamic pair: 輸入 |Q| pair-mag, k=A-n_K
  Global Top-N: 輸入 stage1 score, k=N, 輸出 index → Sparse Gather Engine
```
~2K LUT, 0 DSP。詳細 RTL 與時序待設計 (架構方案確定: streaming sorted register)。

### 4.8 Sparse Gather Engine (attn + FFN 共用 DMA)
```
輸入: index list (動態 runtime)
狀態機: 地址生成 → 發 read request → 收亂序回傳 → 對齊送 PE
  attn Stage 2: index = Global Top-N token, gather K row (補算 full_score)
  FFN         : index = keep neuron, gather W_up[:,keep] / W_down[keep,:]
```
~1.5K LUT, 0 DSP (兩用途分時)。

### 4.9 FFN Threshold Filter Unit
```
proxy[i] = |SiLU(INT8 gate_i)| × W_up_norm[i]    // SiLU LUT × DDR 載入之單層 INT8 範數
動態閾值: 算前幾 token 定及格線 thr; keep = proxy > thr  (非全排序)
跳過 neuron → 其 W_up/W_down row 不從 DDR 載入
```
~0.5K LUT, 0 DSP。W_up_norm 單層 5.6 KB 由 DDR 經 Ping-Pong 載入 (算第 i 層時預取 i+1 層)。

### 4.10 PE Lane / Cross-bar / Softmax / KIVI
- **PE Lane** (QK-PU): active path 8-bit MAC + Adder Tree; SIGN path sign-XOR(Q) 進同一 Adder Tree; 兩者 time-multiplex
- **Cross-bar**: LANE×PE cascading 4:1 mux, pair-mask → active dim, single-cycle
- **Softmax**: base-2 + reciprocal LUT; 輸出 P per-row INT8 量化
- **KIVI** (V-PU): 16×16 systolic INT4×INT8, per-group scale 累加後乘 1 次 (省 dequant multiplier)

### 4.11 Layer 0 異質 mapping
物理 PE = LANE×PE = 48。Layer 0 跑滿 D: ceil(D/48) iteration (D=64→2, D=128→3)。throughput 慢 2-3x, 但 only 1/L layers, 整體 <10%。

### 4.12 Reconfigurability
| 參數 | 範圍 | 控制 |
|------|------|------|
