# Transformer Attention + FFN Co-Sparse Accelerator — 機制與硬體規格 (v7)

**Target**: Xilinx KV260 FPGA (117K LUT, 1248 DSP, 4GB DDR4 64-bit @2400 ≈ 19.2 GB/s peak; 片上 SRAM = BRAM 5.1 Mb + URAM 18 Mb ≈ 23 Mb, 兩種資源分開預算)
**Scope**: LLaMA-family decoder-only Transformer，GQA + SwiGLU + RoPE。**主 model: Llama-3.2-3B-AWQ**; TinyLlama / Qwen2.5 為 cross-model 驗證
**前提**: AWQ INT4 weight-only 量化模型 (premise, 非可選; 硬體 datapath 依賴, 不支援非 AWQ 路徑)

> **本檔為純設計文檔** (機制 / 硬體 / 方法論)。所有實測數字 (PPL / sparsity / union / 精度表 / sweep) 在對應 data md:
> - `tinyllama_data.md` — TinyLlama 校準數據
> - `llama3b_data.md` — Llama-3B (主 model) 校準數據
> - `qwen_data.md` — Qwen cross-model 數據

## 0. 系統總覽

兩個正交稀疏機制 + 全棧量化：

| 區塊 | 機制 | selection 粒度 | 省的資源 |
|------|------|---------------|---------|
| Attention (Q·K·V) | RAD-Cascade + KIVI-V | token-level (active dim + Global Top-N) | Q·K MAC + KV cache BW |
| FFN (SwiGLU) | CETT activation sparsity | neuron-level (真實貢獻 ‖n_i‖ 閾值) | down MAC + down 權重 BW |

兩機制統計獨立 (近線性疊加)，共用 Sparse Gather Engine。

memory-bound 平台 (KV260 單通道 DDR4)：**BW 省 > MAC 省**，設計優先壓 KV cache BW 與權重 BW。

## 0.1 全棧位寬與資料流

原則: **MAC 向量路徑一律整數/精確定點; FP16 只出現在「攤提後標量」或「元素級 O(d) 操作」**。位寬階梯 ablation 數據見 data md。

| 資料 / 運算 | 位寬 | 說明 |
|------|------|------|
| 權重 (所有 linear) | **INT4** (AWQ) + per-group scale (FP16 標量, group=128) | DDR 存 INT4; 不做逐元素 dequant — scale 是 group psum 後標量乘 (1/128 攤提) |
| linear MAC | **FP16-act mantissa-shift × INT4** (Kulisch, 0.2①) | `(m_x × W_INT4) << e_x` → 48-bit 定點累加; per-group psum × s_g 標量。精確 (僅 exp window clamp)。A8 已驗死 |
| 激活 hidden_states | **FP16** | AWQ weight-only 前提; MAC 成本由 mantissa-shift 解掉, 不需 A8 |
| residual + RMSNorm | FP16 (元素級 O(d)) | rsqrt LUT+NR (0.2③); 加法器時分 (0.2④) |
| RoPE | **定點 INT16 sin/cos LUT** (Q1.15) | 旋轉後量化 INT8 |
| Q·K (stage1/2) | **INT8 × INT8** | psum ≤23-bit; SIGN sign-XOR 同樹 |
| K-anchor running-max | FP16 寄存器 | 只比大小 (FP16 正數比較=整數比較, 零 FP 邏輯) |
| KV cache: K | **INT8**; dim-major 副本 + sign plane = long-context 配置 (4.5.2) | |
| KV cache: V | **INT4** (KIVI G=32) | |
| Softmax | **全定點**: 整數 max-sub → exp2 LUT (8-bit fraction) → reciprocal LUT | 無 FP |
| attn_w | **FP16** (aw_mant 11-bit + aw_exp 5-bit signed) | softmax 輸出 ∈[0,1]; aw 長尾 [2⁻⁴⁹,1] 動態範圍大, 定點 (Q0.15) 小 aw 崩 (見 8 dead); FP16 保相對精度 |
| attn_w × V | **FP16 attn_w × U4 Vq** + 係數摺疊, **輸出對齊定點累加** | `out_d = Σ_t (w_t·s_{t,g})·Vq + Σ_t w_t·m_{t,g}`; 不解壓 V; Σaw=1 → 輸出對齊 Q3.20 (小項右移截斷), 非 Kulisch (見 0.2①) |

- 權重 BW: INT4 + group=128 FP16 scale = 4.125 bit/weight → 省 74.2%。
- 全棧位寬合計精度 (各 model PPL) 見 data md。

## 0.2 FP16 必留處與硬體實作 (LUT/DSP)

**① 統一 GEMV lane — mantissa-shift 寬累加 (Kulisch 式)**
```
MAC_i = (m_x × W_INT4) << e_x   → 48-bit 定點累加器
  m_x = FP16 mantissa (11-bit, 含 hidden bit), e_x = exponent
```
- **精確運算**: mantissa×INT4 整數積移位後寬累加無捨入, 比 FP16 FMA 鏈更準; 唯一近似 = exponent window clamp
- 成本/lane: 乘法 11×4-bit → 27×18 DSP **可 2-pack** (~0.5 DSP) + barrel shifter ~30 LUT + 48-bit 累加 (DSP48E2 ALU 恰 48-bit, 或 fabric)
- exponent window [2⁻¹², 2¹³]: 離線校準激活 exponent 直方圖定窗 (3.1 流程), **飽和計數器** runtime 監控。實測覆蓋率 / clamp 代價見 data md (clamp_hi=0, clamp_lo 良性)
- V mode: 同一 27×18 乘法器吃 ews_mant(=aw_mant×m_s, 22-bit) × U4(Vq); **但累加對齊與 linear 不同** — attn·V 有 Σaw=1 + 輸出有界 (≈1.1), 用**輸出對齊定點** (各項右移 |aw_exp+e_s| 到 Q3.20, 小 aw 項截斷), 非 Kulisch 對齊最小項。乘法器 + 物理 acc (48-bit) 共用; shift 方向 (linear 左移對齊最小 / V 右移對齊輸出) + 累加模式 mux 切換。
  > **為何 attn·V 不能用 Kulisch**: aw 是 softmax 長尾 ([2⁻⁴⁹,1]), Kulisch 對齊最小項 → acc 撐爆 80-bit。輸出對齊利用 Σaw=1 (小項 < ULP 自然截斷), acc 24-bit。實測見 data md。

**② scale 標量乘 (per-group s_g, per-token a_s, KIVI c_{t,g})** — 同 ① trick: `psum(INT) × m_s << e_s`, 攤提 1/128 (權重) / 1/32 (V), 共 ~4-8 個此類乘法器全晶片。

**③ rsqrt (RMSNorm)**: LUT 2⁸×16b + 1 次 Newton 迭代, 1 DSP, per-token per-norm 1 次 (O(1))。

**④ residual FP16 加法器**: ~70 LUT/個 × 8 個時分。

**⑤ exp2 / reciprocal LUT (softmax)**: BRAM 2⁸×16b 各一, 0 DSP。

## 1. 機制 (model-agnostic)

### Notation

| Symbol | 定義 |
|--------|------|
| `L` | 層數 |
| `H_Q` / `H_KV` | Query / KV head 數 (GQA) |
| `D` | head_dim |
| `P` | num_pairs = D / 2 |
| `S` | sequence length |
| `A` | active pairs per query |
| `n_K` | K-anchor pair 數 |
| `N` | Global Top-N candidate 數 |
| `G` | KIVI V group size |
| `d_ff` | FFN intermediate size |
| `τ_CETT` | FFN CETT 閾值 (PPL+1% 標準對應) |
| `ε_i` | 第 i 層 FFN neuron 貢獻閾值 (τ_CETT 反推, 離線校準) |
| `G_Q` | GQA group 大小 = H_Q / H_KV (共用同一 K 的 Q head 數) |

### Layer-wise dispatch

| Layer | Attention | FFN |
|-------|-----------|-----|
| Layer 0 | 異質: 跑滿 D dim (active=P, n_K=0), INT8 Q·K + INT4 V, 不套 RAD selection | CETT activation sparsity |
| Layer 1 ~ L-1 | RAD-Cascade (active + SIGN + Global Top-N) + KIVI-V | CETT activation sparsity |

Layer 0 異質原因: per-row attention 分散度高 (max ≈ 1/S), RAD selection 不適用; 跑滿維度, 後續層才省。

### 1.A Attention: RAD-Cascade

量化: Q, K per-token INT8 (進 PE 後轉)。

#### Stage 1: Active path + SIGN 補償 (Q·K 端)

**Active dim 選擇** (INT8 MAC, full-D-dim einsum with mask):
- 每 query 選 `A` 個 active pair，兩類組成：
  - `n_K` 個 **K-anchor pair**: per-position streaming running-max of |K| (FP16) 選 top-n_K (見 2.7)
  - `A - n_K` 個 **dynamic pair**: **GQA shared-dim** — per KV-head group (共用同一 K 的 G_Q 個 Q head) 以 `Σ_h |Q_h|` pair-mag 選 top, anchor scatter 排除; 組內所有 Q head 用同一組 dim。**選擇輸入用量化前 FP16 幅度** (selection 是純比較, FP16 比大小=整數比較, 零成本), MAC 仍 INT8
- **為何 shared 而非 per-head (GQA union 效應)**: K 每步 per KV head 只讀一次, per-head 各選 dim 時實際讀取 = G_Q head 的 dim union → K BW 省從單 head 上限 (1-A/P) 塌縮。shared-dim 把 union 收斂回 A dim, MAC 省不受影響 (per-head 本來就只乘自己的 dim), shared-dim 是純 BW 修正。union 實測值與 shared-dim 端到端代價見 data md。
- pair → dim: `full_dim_mask = pair_mask.repeat_interleave(2)`
- `active_score = einsum(Q_b ⊙ full_dim_mask, K_b)` (mask 乘法後全 D 維 einsum, 等價只算 active 維)
- 不使用 Q-anchor (n_Q = 0, 見 2.5)

**SIGN 補償** (純 sign, 1-bit sign(K) + 8-bit |Q| 累加, 零乘法器):
```
sign_k = (K_b >= 0) ? +1 : -1
comp   = einsum(Q_b ⊙ dropped_mask, sign_k)    # Σ_d∈dropped  q_d · sign(k_d)
stage1 = active_score + β × comp
```
- β = 1.0 (Theorem 1)
- **純 sign-only, 無 mean(|K_drop|) 縮放** (sign-XOR 直接進 adder tree, 不需 |K| 統計表)

**Theorem 1** (SIGN unbiased): 對稱分布下 E[sign(k)·k] = mean(|k|), β=1.0 為 unbiased estimator。

#### Stage 2: Global Top-N selection (prefill soft-tail / generation hard-truncation)

```
full_score    = einsum(Q_b, K_b)                       # 全 D 維 INT8 (候選用)
stage1_masked = where(valid_causal, stage1, -inf)
topN_idx      = topk(stage1_masked, N)                  # row-global top-N (per-head)
cand_mask     = scatter(topN_idx) & valid_causal
# prefill : final_score = where(cand_mask, full_score, stage1)   [soft-tail, 讀全 V]
# generation: final_score = where(cand_mask, full_score, -inf)   [hard-trunc, V 讀 per-KV-head union]
```

**prefill 與 generation 分開處理 V (實測決定):**

| 階段 | 非候選 score | V 讀取 | 理由 |
|------|-------------|--------|------|
| **prefill** | 保留 stage1 近似 (soft-tail) | 讀全部 V | hard-trunc 崩 (見下) |
| **generation** | -inf (hard-truncation) | 只讀候選 V (per-KV-head **union**) | 可行, 省 V BW (GQA union 真值見 data md) |

- **prefill 必須 soft-tail**: prefill 全 query 同時 hard-truncation → PG-19 PPL 崩潰 (見 dead directions; S 個 query 表徵砍到只剩 N target, 累積毀掉)。
- **generation 可 hard-truncation**: 每步單 query 對歷史 KV 選 top_n, 非候選 -inf 只讀候選 V, 不崩 (XSum ROUGE drop 見 data md)。同 SparQ Step2 (decode 只 fetch top-k V), prefill 不做。
- **GQA 真值 (top-N 維持 per-Q-head, 讀取按 union)**: V per KV head 只讀一次, 實際讀取 = G_Q 個 Q head 候選集的 union。union 實測值 + V BW 省見 data md (非 per-head 帳面的理論上限)。
- **shared top-N 已驗死** (見 dead directions): 強制共用候選 → 8 head 候選本質分歧, PPL 崩。**候選選擇保持 per-head, BW 帳按 union 算**。
- **與 SparQ 異同**: top-k token 選擇 + dim 近似框架同 SparQ (公開方法, 已引用); 增量為 SIGN 補償 (stage1)、K-anchor running-max、**GQA shared-dim selection** (per-head 在 GQA 下 BW 省塌縮, SparQ 未處理)、INT8/INT4 量化整合、FPGA MAC 省。
- `N = f(S)` (見 1.C)；row-global，非 per-block 配額。

#### Stage 3: KIVI-V + attn_w

- attn_w (softmax 輸出): **FP16** (aw_mant 11 + aw_exp 5; base-2 exp 天然 mantissa+exp, 不壓定點)
- V: per-token group-wise INT4 (KIVI, Liu et al. 2024), default `G = 32`
- **attn_w × V**: FP16 attn_w × U4 Vq + 係數摺疊, 輸出對齊 Q3.20 累加 (不解壓 V, 見 0.1/0.2①)
  - **prefill**: 讀全部 V (soft-tail 需全 V 參與)
  - **generation**: 只讀候選 V (非候選 softmax=0)
```
For each group g of G channels:
    scale_g = (max-min)/15;  V_q = round((V-min)/scale_g) ∈ [0,15]
```
- **V BW 省**: prefill 靠 INT4 壓縮 (~75%); generation 額外靠候選截斷 (讀 per-KV-head union)。

### 1.B FFN: CETT Activation Sparsity

#### 原理
SwiGLU: `out = W_down( SiLU(x·W_gate) ⊙ (x·W_up) )`。每 neuron 對 MLP 輸出的真實貢獻向量 `n_i = h_i · W_down[:,i]` (h_i = SiLU(gate_i)·up_i)。跳過低貢獻 neuron → 其 W_down[:,i] 不算/不載 → 省 down MAC + down 權重 BW。

#### Selection: CETT ε 閾值 (training-free, 動態)
判據用 neuron **真實貢獻範數** (含 down 投影, 比純 gate proxy 準):
```
‖n_i‖ ≈ |h_i| × ‖W_down[:,i]‖           # h_i 標量 × down 列範數 (SRAM 常數)
跳過 ‖n_i‖ < ε_i 的 neuron               # 動態, 砍量隨輸入/token 變
```

**CETT 度量** (TDA Eq.20, 校準 ε 用):
```
D = { i | ‖n_i‖ < ε }
CETT = ‖ Σ_{i∈D} n_i ‖₂ / ‖ MLP_output ‖₂     # 被砍 neuron 輸出向量和範數 / 完整輸出範數
```
- 分子為**向量和的範數** (有方向抵消)，非標量平方和。標量近似差為 per-model (見 data md)，定義以 TDA Eq.20 為準。

#### 標準: PPL 漲 1% (文獻泛化, 非 overfit 固定比例)
- τ_CETT = 使 validation PPL 漲 ~1% 的 CETT 值 (Sparsing Law arXiv 2411.02335 證明此標準泛化性最強)。
- 每層 ε_i 由 τ_CETT 反推 (二分搜，離線校準)。
- **τ_CETT 是 model-agnostic 標準**; sparsity 是結果 (隨 model/輸入自適應)，不寫死。per-model 落點見 data md。

#### prefill 與 generation 統一動態
- prefill 與 generation **都用 CETT ε 動態砍** (每 token 算 ‖n_i‖ 比 ε_i)。
- **不使用固定 mask 複用** (TDA-style): generation 固定 mask 與動態需求重合度低 (見 dead directions), 固定 mask 本質不適用。

#### 機制邊界 (為何只省 down)
training-free + 不改激活函數 + SwiGLU 三約束下，**只能省 down**:
- 要選哪些 neuron 砍，須先算 h_i = SiLU(gate)·up → **gate/up 必須全算** (mantissa-shift lane, 不省)。
- SwiGLU (SiLU) 無硬零 (不像 ReLU)，激活不可廉價預測 → 無法免訓練 predictor 預測 mask 省 gate/up。
- 文獻中省 gate/up 者皆需 (a) train predictor (DejaVu/PowerInfer) 或 (b) 改激活為 ReLU²/FATReLU 並續訓 (ProSparse) → 違反 training-free + no-modification。
- TDA 論文亦明言「only down projection reduced <50%」→ 只省 down 是約束的數學本質。

#### CETT vs 舊 gate-proxy (為何升級)
- gate proxy (|SiLU(gate)|×‖W_up‖) 上限低 (gate⊥up 統計獨立)，砍 25% 即崩。
- CETT (含 down 投影 ‖n_i‖) + ε 動態 + PPL+1% 標準 → 無損砍量翻倍且更嚴謹 (具體 sparsity 見 data md)。

### 1.C 序列級自適應 (active 與 top_n 隨 S 配置)

attention 兩稀疏參數依**序列長度 S** 在 prefill (S 已知) 配置一次，**整段固定** (非 per-query → MAC 陣列配置一次, 無 pipeline 氣泡)。

**active: 固定 37.5%**:
- 「短序列降 25% 省 MAC」與數據矛盾 (短序列精度餘裕更小; 且短序列 Q·K MAC 占比極低, 無省價值)。刪雙檔, 固定 37.5%。
- active dim 須 DMA 對齊; per-model 校準只定比例 (25/37.5/50% 三 candidate)。

**top_n=f(S)** (四檔):
| S | top_n |
|---|-------|
| S < 128 | 8 |
| 128 ≤ S < 256 | 16 |
| 256 ≤ S < 1024 | 32 |
| S ≥ 1024 | **64** |

- 長 S 加 64 檔: N sweep 證明 N=64 邊際遞減停點, attention 全棧壓進 +1% 標準 (sweep 數據見 data md)。S=1024 切點為內插。
- top_n 影響 Stage 2 gather 候選數 (gather 層面可變, 不破壞 MAC 陣列)。

**為何序列級固定而非 per-query / per-layer**:
- per-query 動態 active → MAC 陣列每 query 重配 → 氣泡。
- per-layer 差異化 active 無 model-agnostic 免校準判據。
- top_n 動態為 gather 層面, 不觸 MAC 陣列, 隨 S 配置無氣泡。

### 完整 dataflow
```
Per layer:
  RMSNorm (FP16, rsqrt LUT+NR)
  [Attention]
    QKV 投影: x(FP16) @ W_qkv(INT4): mantissa-shift lane, per-group psum × s_g → Q/K/V
    RoPE (INT16 sin/cos LUT) → Q,K → 量化 INT8
    K: |·| 更新 running-max 錨點 (片上); INT8 寫 cache (row-major; dim-major 副本依 long-context reg)
    V: 量化 INT4 寫 cache
    Per query (generation: per step):
        1. K-anchor: top-n_K of 當下 K_runmax
        2. Dynamic : per KV-head group Σ_h|Q_h| pair-mag 選 top-(A-n_K) (shared-dim, anchor 排除)
        3. Stage 1 : active_score (INT8) + β·SIGN (sign-XOR, 零乘法)
        4. Stage 2 : full_score(INT8) on per-head top-N candidate
                     prefill: final = where(cand, full, stage1)  [soft-tail, 讀全 V]
                     generation: final = where(cand, full, -inf)
                     generation gather: G_Q-head 候選 union → 單一 index stream 同時取 K row + V row
        5. Softmax (max-sub → exp2 LUT (保留 mantissa+exp) → reciprocal) → attn_w (FP16: aw_mant+aw_exp)
        6. attn_w(FP16) × Vq(U4) 係數摺疊, 輸出對齊 Q3.20 累加 → attn_out (不解壓 V) → FP16 (給 o_proj)
    o_proj: attn_out @ W_o (mantissa-shift lane); residual add (FP16)
  [FFN]
    RMSNorm (FP16)
        7. gate = x·W_gate, up = x·W_up   (mantissa-shift lane, 全算)
        8. h_i = SiLU(gate_i)·up_i (SiLU LUT)
        9. ‖n_i‖ = |h_i| × ‖W_down[:,i]‖(SRAM); keep = ‖n_i‖ ≥ ε_i  (動態)
       10. down: gather keep 的 W_down column → ffn_out; residual add (FP16)
```

## 2. 軟體超參方法論

> 具體校準數值 (β / G / active ratio / n_K / N / τ_CETT 落點) 為 per-model, 見 data md。本節只記**如何測得**。

### 2.1 Metric Hierarchy
| Metric | 用途 | 可信度 |
|--------|------|-------|
| Spearman ρ | 僅 active dim ratio sweep | 機制存廢預測失效, 不可作機制決策 |
| KL max | n_K sweep | 跨 model 驗證可信, primary |
| CETT (PPL+1%) | FFN ε 校準 | 文獻泛化標準 |
| End-to-end (HellaSwag/PG-19/XSum/LongBench) | ground truth | 任何機制變動必須驗 |

**「最優」紀律**: 任何「最優」宣稱須附對照上界。

### 2.2 β (SIGN) = 1.0 (Theorem 1 + sweep)
### 2.3 KIVI group G = 32 (KIVI default)
### 2.4 active_dim ratio A/P — 固定 37.5% (雙檔已刪, 見 1.C); 新 model 三 candidate (25/37.5/50%) + end-to-end
### 2.5 不使用 Q-anchor: n_Q=0 (1Q vs 0Q gap noise floor; SparQ cross-model 皆無; register 保留 fallback)
### 2.6 K-anchor 數量 n_K ∈ {1,2,3,4} per-model (KL max → end-to-end)
### 2.7 K-anchor 選法: per-position streaming running-max
```
Init K_runmax[L,kv_h,p]=0
Per new K_i: K_runmax[p] ← max(K_runmax[p], max(|K_i[2p]|,|K_i[2p+1]|))   # FP16 片上
Per query:   K_anchor = top-n_K of 當下 K_runmax
```
驗證: runtime running-max 勝 offline 校準 (long context 自動抓 attention sink); 三估計法差 < noise floor。採 per-position (硬體最簡: P 維 FP16 running-max 寄存器, 零 offline)。

### 2.9 FFN: CETT 閾值 τ_CETT (PPL+1% 標準)

### 2.10 GQA shared selection 方法論
- **K dim union counter**: 統計 per-KV-head group 內 G_Q 個 Q head 各選 active pair 的 union 大小 (vs 獨立上界)。union 接近上界 → per-head BW 省塌縮 → shared-dim 必要。
- **shared-dim 驗證**: per-KV-head Σ_h|Q_h| 選共用 dim, 測 e2e PPL vs per-head。代價 noise floor 內 → 採用。
- **shared top-N 驗證**: 8 head max 合併選共用候選, 測 e2e PPL。崩 → 死 (token 候選 head-specific)。
- **V token union counter**: per-head top_n 的 G_Q-head token union → V BW 真值 = 1 - union/S。
- **非對稱 insight**: dim 重要性結構性共享 (shared 幾乎無損); token 重要性 head-specific (不能共享)。
- 跨層確認 (一淺一深層) 確保結論穩定。實測值見 data md。

### 2.11 位寬階梯 (累加 ablation 方法論)
- L0 shared-dim base (FP16) sanity → L1 +RoPE INT16 → L2 +定點 softmax/U16 → (A8 分解驗死)。
- 每級對前級增量必須實測; A8 分解到 per-linear (qkv/o/gate/up/down) 找主因。
- 具體 PPL 階梯見 data md。

### 2.12 selection FP16 + top-N sweep 方法論
- sel_fp16 (選擇輸入量化前 FP16): 零硬體成本 (純比較), 對症 RAD 選擇對 Q/K 擾動敏感。
- top-N sweep 看邊際遞減停點 + V union 增長。數據見 data md。

### 2.13 exponent window 校準方法論
- 統計各 linear 輸入激活 e_x = floor(log2|x|) 直方圖 → 定 mantissa-shift lane window。
- 確認 clamp_hi (上界 outlier) / clamp_lo (下界 underflow) 覆蓋率; clamp PPL 代價實測。
- 下界 clamp (極小值歸0) 對 MAC 無害; 上界 outlier 才致命。實測見 data md。

```
├── Sparse Gather Engine                ── attn Stage2 K+V (單 index stream) / FFN down neuron 共用 DMA
├── CETT Filter Unit                    ── FFN ‖n_i‖ vs ε_i 動態裁決
├── Weight Dequant Unit                 ── INT4 → FP16 (per-group scale), 餵 GEMV Engine
└── Memory (K/V/Q Buffer [URAM]; K_runmax + 小表 [BRAM]; K 雙 layout + sign plane + W_down_norm + AWQ weights [DDR4])
```

### 4.5 硬體資源共用

**① 統一 Top-K Engine (≤64) — 三 stage 共用**
K-anchor (k≤4)、Dynamic pair (k≈A-n_K)、Global Top-N (k=N≤64) 皆「分數流選 top-k」，k≤64 且時間不重疊。共用 64-entry sorted shift register。

**② FFN selection 不併入 Top-K Engine — 獨立 CETT Filter**
FFN 留 neuron 數巨大 (千級)，streaming 全排序面積爆炸。改用 **CETT 閾值過濾** (‖n_i‖ vs ε_i, 過線即留)，非全排序。

**③ Sparse Gather Engine — attn Stage 2 與 FFN down 共用**
Stage 2 token gather 與 FFN down neuron 權重 gather，狀態機相同 (地址生成 → read request → 收亂序回傳)。共用同一 DMA gather 控制器。

**④ Adder Tree — Active MAC 與 SIGN 共用**
SIGN = Σ|Q_d|·sign(K_d): sign(K) 對 Q 做 XOR 取反 (不需乘法器), 進 active path 同一 Adder Tree。time-multiplex 共用。

**⑤ Weight Dequant Unit — 所有 linear 共用**
INT4 + per-group FP16 scale → FP16。register-level (送進 MAC 前一刻 dequant), 不增 DDR BW。

**⑥ Linear/V GEMV Engine — 所有 linear 與 attn_w·V 共用 (取代獨立 V-PU)**
generation (batch=1) 下所有 linear 都是 GEMV: `x@W = Σ_i x_i × W[i,:]` (scalar × row 累加); `attn_out = Σ_t attn_w_t × V[t,:]` 是**同一 dataflow** (標量 × INT4/U4 row 寬累加)。一套 mantissa-shift 寬累加引擎分時服務兩者, 差異只在標量/row 來源: linear = m_x(FP16 mantissa) × W_INT4, Kulisch 左移對齊最小 (48-bit); V = ews_mant(aw_mant·m_s) × Vq_U4, 輸出對齊右移截斷 (Q3.20, Σaw=1, 24-bit)。乘法器+物理acc共用, shift方向/累加對齊兩模式 (係數摺疊不解壓 V, 見 0.2①)。砍掉獨立 V-PU systolic, 省 ~96 DSP + 一套 FSM。generation BW-bound, MAC 大量閒置, 共用零吞吐代價。
> 舊 spec「QK-PU 與 V-PU 不共用」論證只對 QK 成立 (scalar-reduction vs vector-accumulate 正交), 對 V-PU vs linear 不成立 (兩者同為 vector-accumulate); 且舊資源表沒給 linear (占總 MAC 73-78%) 任何運算單元 — 此為前版最大漏洞。

**不共用: QK-PU 與 GEMV Engine** — reduction 方向正交 (QK 歸約成 scalar; GEMV 跨 row 累加成 vector), 位寬不同 (INT8 vs FP16)。兩套獨立 datapath。

### 4.5.1 序列級自適應配置單元 (top_n = f(S))
```
At prefill (S known):
    top_n = (S<128)?8 : (S<256)?16 : (S<1024)?32 : 64   // active 固定 37.5%
```
純比較器, ~0 額外 DSP, 序列開始配置一次 (無氣泡)。

### 4.5.2 K cache 雙 layout (DDR burst 對齊, K BW 省成立的前提)

**問題**: K row INT8 = 恰好一個 DDR4 burst。row-major 單份下「只讀 active dim」仍觸發整 burst → dim-skip BW 省 = 0。

**自攻過的替代 (皆死, 見 dead directions)**: 單份 dim-major (stage2 散讀吃掉省的); block-transpose tile (tile 需大才滿 burst, stage2 慘)。

**採用: 雙 layout (SparQ 原文同做法, store K twice)**:
| 份 | layout | 服務 | 讀法 |
|----|--------|------|------|
| K_dim | dim-major INT8 (每 dim 連續 S byte) | generation stage1 | shared-dim active dim × S, 連續 burst |
| K_sign | dim-major 1-bit packed (D×S bit) | SIGN 補償 | dropped dim × S/8 byte |
| K_row | row-major INT8 (每 token = 1 burst) | stage2 gather + prefill | 候選 union × row |

- 成本: K 有效 2B/元素 (寫兩次, generation 每步只寫 1 token ×2, 可忽略); KV cache 存儲省下降 — 用存儲換真實 BW 省。具體 byte-level K BW 省見 data md。
- **prefill scope**: prefill tile 內多 query 各自選 dim, union 趨近全維 → K 全讀, **prefill K dim-skip BW 省 ≈ 0 (本質)**。dim-major 份純為 generation 服務。

### 4.5.3 Prefill tiling (URAM 約束)

S 大時單層 KV > buffer → 必須 tile。骨架 (per KV head, 帶其 G_Q 個 Q head):
```
Q tile = Q_tile position × G_Q Q head × row 駐留
KV 沿 S 串流 (兩遍式):
  pass 1: stage1 (K_dim 串流) + per-head Top-N 選擇 + online softmax 統計 (max / Σexp on stage1)
  pass 2: 候選 full_score 修正 (K_row gather) + attn_w·V 累加 (V 串流, soft-tail 全 V)
K 重讀因子 = 2 × S / Q_tile
```
- prefill K/V 流量 ∝ S²/Q_tile, Q_tile 受 URAM 上限; 精確 Q_tile 與 pass 合併為 RTL 階段定案項。
- prefill 為一次性成本, demo 主指標 (tok/s) 由 generation 決定; 先鎖架構不鎖數字。

### 4.6 K-runmax update unit (per-position streaming, FP16)
```
Per K_i vector:
    Step1 pair abs-max:  K_pair[p] = max(|K_i[2p]|,|K_i[2p+1]|)   // P 並行 FP16 compare
    Step2 running-max :  K_runmax[p] ← max(K_runmax[p], K_pair[p])
Per query: 送當下 K_runmax 給 Unified Top-K Engine 選 top-n_K
```
~0.5K LUT, 0 DSP, 1 cycle/K vector。FP16 (只比大小)。

### 4.7 Unified Top-K Engine (≤64)
```
64-entry sorted register (score 16-bit + index), k 由 reg 配置:
  每新分數 s: if s > min_entry: 插入冒泡淘汰最小   // O(k) compare, 1 score/cycle
三 stage 分時: K-anchor (k=n_K) / Dynamic (k=A-n_K) / Global Top-N (k=N → Gather)
```
~4K LUT (64-entry), 0 DSP。

### 4.8 Sparse Gather Engine (attn + FFN 共用)
```
輸入: index list (runtime)
狀態機: 地址生成 → read request → 收亂序回傳 → 對齊送 PE
  attn generation: index = G_Q-head 候選 union → 單一 index stream 同時 gather K_row (full_score) + Vq row (attn·V, U4 不解壓)
  FFN down       : index = keep neuron → gather W_down column
```
**W_down 必須轉置存儲 [d_ff][hidden] (column-major)**: 標準 [out][in] 下一個 column = 散落 nibble, gather 全廢; 轉置後 column 連續, burst 友好。AWQ scale 沿 in-dim 分組, 散砍下幾乎每 group 都被觸碰 → scale 全讀 ≈ down 權重 3%, 可接受。
~1.5K LUT, 0 DSP (分時)。

### 4.9 CETT Filter Unit
```
‖n_i‖ = |SiLU(gate_i)·up_i| × W_down_norm[i]   // gate/up FP16 (GEMV 輸出); SiLU LUT; 範數 per-group(128) INT8
keep = ‖n_i‖ ≥ ε_i   (ε_i per-layer 常數, SRAM)      // 動態, 非全排序
跳過 neuron → 其 W_down column 不從 DDR 載入, down MAC 不算
```
~0.6K LUT, 0 DSP。

### 4.10 PE Lane / Cross-bar / Softmax / GEMV / Dequant
- **PE Lane** (QK-PU): active INT8 MAC + Adder Tree; SIGN sign-XOR(Q) 進同一 Adder Tree; time-multiplex
- **Cross-bar**: LANE×PE cascading mux, pair-mask → active dim, single-cycle
- **Softmax**: base-2 exp (保留 mantissa+exp) + reciprocal; 輸出 attn_w FP16 (aw_mant 11 + aw_exp 5)
- **Linear/V GEMV Engine**: 128 lane; 共用乘法器 (27×18) + 物理 acc (48b)。linear mode: m_x×W_INT4, Kulisch 左移, 48b。V mode: ews_mant(aw_mant·m_s)×Vq, 輸出對齊右移截斷, Q3.20 (24b)。shift方向/累加對齊 mux 切換, 不解壓 V
- **Weight Dequant**: INT4 + per-group FP16 scale → FP16, 餵 GEMV

### 4.11 Layer 0 異質 mapping
物理 PE = LANE×PE = 48。Layer 0 跑滿 D: ceil(D/48) iteration。throughput 慢 2-3x, only 1/L layers, 整體 <10%。

## 5. 各機制節省定位 (方法論)

> 具體 counter 數字 (MAC 省 % / BW 省 % / 精度表) 為 per-model, 見 data md。本節只記**定位與口徑**。

### 5.1 MAC 省定位
- **誠實定位**: 你的機制 (RAD + CETT) 省 **MAC**; BW 省主要靠量化 (AWQ 權重 + KIVI KV cache, 引用非原創)。
- 分機制各自對自己 block 的 dense baseline (不混算整體, 因 FFN 占 MAC 73-78% 會稀釋 attention)。
- Attention Q·K MAC 省上限 = 1 - active_ratio; stage2 top_n + layer0 dense 拉低。
- **GQA shared-dim 不改 MAC 數字** (per-head 本來只乘自己的 dim); 它修的是 K 讀取 BW。
- FFN: down 占 FFN MAC 1/3, gate/up 全算 → FFN 整體 MAC 省 ≈ sparsity/3。
- **MAC 占比**: FFN linear 占總 MAC 73-78% (大頭); Q·K 占比小 (長序列才上升)。整體 MAC 省主力 = CETT FFN; attention RAD 在長序列 + 精度 + memory-bound BW 才顯價值。

### 5.2 BW 省定位 (generation per step, GQA 真值口徑)
| 資源 | 機制 | 歸屬 |
|------|------|------|
| 權重 (所有 linear) | AWQ INT4 (4.125 bit) | AWQ 引用 |
| KV cache 存儲 | K INT8×2 + sign plane + V INT4 | KIVI 引用 + 你的 layout |
| K 讀取 (generation) | shared-dim active dim (dim-major) + sign plane + union gather | 你的 (GQA shared-dim) |
| V 讀取 (generation) | per-head top-N, 讀 union | 你的 (SparQ式截斷 + GQA union 帳) |
| FFN down 權重 | 砍 sparsity neuron (W_down 轉置 column gather) | 你的 (CETT) |

**Roofline 誠實定位**:
- generation 瓶頸 = **權重 BW** (每步讀全部權重 ≫ KV cache)。KV 端全部優化只動 generation 流量小部分。
- generation tok/s 上限由權重 BW 決定 (AWQ 74% + CETT down 已到 training-free 邊界)。
- K/V 端機制的價值: **long-context 縮放** (KV ∝ S, 長 S 時 K 與權重同量級, 省幅才放大) + **精度框架** (RAD 是精度損失主要承擔者)。短序列 demo 不靠它撐 tok/s, 不誇大。

### 5.3 精度代價口徑 (真 AWQ 部署, 4-way 分離)
- **分離量化損失 (AWQ vs FP32) 與機制損失 (機制 vs AWQ)**: 機制在真 AWQ 部署額外損失小, additive 不放大。
- generation V 候選截斷 vs soft-tail: ROUGE 多掉少量, 換 V BW 省大。
- 口徑: per-head vs shared-dim 分開報; FP32 口徑 vs AWQ 口徑分開。具體數字見 data md。

## 8. Spec Discipline

**系統級精度 budget**: attention 機制 (RAD 全棧含量化位寬) 目標 ~+1% 級, **報實測不設硬線**; 不為硬壓 +1% 犧牲 active 比例 (省比例優先); FFN 機制 (CETT) PPL+1% 標準; **系統合計宣告 ≤ +2.5%** (兩機制近加性)。避免「per-mechanism 各守標但合計看似超標」的口徑歧義。
**Chunk 清單口徑**: PG-19 chunk 篩選以 dense 跑一次**凍結清單**, 所有配置共用 — 禁止 per-run 獨立篩選。
**子集基線紀律**: 任何子集 (nc<full) 選擇用 sweep, δ 必須除以**同子集** dense 基線。
**負 δ assert**: sweep 腳本內建 δ < -0.3% 即 abort (「PPL 降低必為 bug」自動化)。
- **機制決策流程**: L1 ρ filter → L2 KL max(attn)/CETT(FFN) → L3 random 500 end-to-end → L4 full HellaSwag + PG-19 + XSum/LongBench
- **約束驗證**: 每約束須物理硬體理由
- **Regression**: 已驗證 *_final.json 直接引用不重跑
- **Sanity sample**: random/stratified (非前 N, 避 bias)
- **Methodology**: 機制變更須 short+long context; 「最優」宣稱須附對照上界; 物理不可能數字 (如 PPL 降低) 必為 bug
