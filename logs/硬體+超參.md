# Transformer Attention Accelerator - 機制與硬體規格

**Target**: Xilinx KV260 FPGA (117K LUT, 1248 DSP, 4GB DDR4, 640 KB BRAM)
**Scope**: LLaMA-family decoder-only Transformer (TinyLlama, Qwen, LLaMA2, ...)

---

## 1. 機制 (model-agnostic)

### Notation

| Symbol | 定義 |
|--------|------|
| `L` | 層數 (model-dependent) |
| `H_Q` | Query head 數 |
| `H_KV` | KV head 數 (GQA) |
| `D` | head_dim |
| `P` | num_pairs = D / 2 |
| `S` | sequence length |
| `R` | GQA ratio = H_Q / H_KV |
| `G` | KIVI V group size |

### Layer-wise dispatch

| Layer | 處理機制 |
|-------|---------|
| Layer 0 | 異質管線, 跑滿 D dim, 8-bit Q·K + 8-bit V (不套 RAD) |
| Layer 1 ~ L-1 | RAD-Cascade (Q·K) + KIVI-V (V) |

Layer 0 為什麼異質: attention 分散度高 (per-row max ≈ 1/S), RAD 機制不適用. 此 layer 跑滿維度後續層才省 MAC.

### RAD-Cascade Stage 1: 主動 + 補償 (Q·K 端)

**Active path** (8-bit MAC):
- `A` active pairs per query (calibrated, default `A = round(0.375 × P)`)
- 由三類 pair 組成:
  - `n_Q` 個 static Q-anchor pair (per-(layer, head), offline calibrated)
  - `n_K` 個 static K-anchor pair (per-(layer, head), offline calibrated)
  - `A - n_Q - n_K` 個 dynamic pair (per-query top-K by |Q|)
- 約束: `n_Q + n_K ∈ {1, 2, 4, 8}` (hardware-aligned 2^n)

**SIGN compensation path** (1-bit XNOR + 8-bit scale):
- 對其他 `P - A` 個 dropped pair (`D - 2A` dim) 算:
- `score_comp = β × mean(|K_drop|) × Σ_d (|Q_d| × sign(Q_d) ⊕ sign(K_d))`
- β = 1.0 (Theorem 1 unbiased optimal, 已跨 model 驗證)

**Theorem 1** (SIGN Compensation Unbiased):
取期望 E[sgn(K) × K | K symmetric] = mean(|K|), 故 β=1.0 為 unbiased estimator.

### RAD-Cascade Stage 2: Block dispatch

對每個 query, 把 K-side 切成 `B` block (`B = 16` default)

**Per block**:
- `block_max`, `block_mean` 算出來
- `signal_amp = block_max - block_mean`
- `margin = top1 - top2` of block scores

**Dispatch rule**:
- `bypass`: `margin > signal_amp × T_margin` → 只送 top-1 candidate
- `overflow`: `score ≥ block_max - signal_amp × T_overflow` → 候選, 最多 `M_local` 個 / block

**Default thresholds** (Chebyshev confidence interval derived):
- `T_margin = 0.75` (3σ confidence, P(error) ≤ 11%)
- `T_overflow = 0.25` (1σ tolerance, cover 70% candidates)
- `M_local = 3`

**Candidate 補算**: 對 candidate token 補算 dropped `D - 2A` dim 用 8-bit MAC.

### KIVI-V (V 端 quantization, 取代原 PxV)

對每個 V token 做 **per-token group-wise INT4 quantization** (Liu et al., KIVI 2024).

**演算法**:
```
For each (B, H, S) token:
    For each group g of G consecutive V channels:
        v_min = min(V_token[g*G : (g+1)*G])
        v_max = max(V_token[g*G : (g+1)*G])
        scale_g = (v_max - v_min) / 15
        V_q[g] = round((V[g] - v_min) / scale_g)  ∈ [0, 15]   (INT4)
    Store (V_q, scale_g, v_min_g) per group
```

**Default**: `G = 32` (KIVI paper default)

**Why over PxV** (probability-based split):
- PxV 用 `attn > P_THRES` 切分被 ZipCache (NeurIPS 2024) 論證為不準確的重要性 metric
- KIVI 用 per-token group-wise outlier handling, **no threshold needed**
- Equal accuracy (HellaSwag drop 1.04 pp), 但更高 V MAC saving (47.73% vs 33.64%)

### 完整 dataflow

```
1. Q·K Stage 1 (active + SIGN): D/A ratio MAC saving
2. Q·K Stage 2 (block dispatch): 候選補算
3. Softmax → attn_prob
4. Group-wise dequantize V_int4 with scale_g
5. attn_prob @ V_dequant → output (with scale-extraction累加)
```

## 2. 軟體超參如何測得 (model-agnostic methodology)

### 2.1 β (SIGN compensation coefficient)

**值**: 1.0 (Theorem 1 unbiased)
**方法**: Theorem 推導 + 200 sample sweep (β ∈ {0.3, 0.5, 0.7, 1.0}) 驗證

### 2.2 KIVI group size `G`

**值**: 32 (KIVI paper default)
**方法**: 引用 KIVI paper, 不重 sweep
**Note**: G 太小 (≤ 16) metadata 開銷大, G 太大 (≥ 128) 量化誤差累積. G=32 是 KIVI 已驗證的 sweet spot.

### 2.3 active_dim ratio `A/P`

**方法**: Spearman ρ sweep on representative layers (排除 Layer 0)

**演算法**:
```
Select 4-5 representative layers (排除 L0)
For each ratio in {12%, 18%, 25%, 31%, 37%, 50%, 62%}:
    For each layer L in selection:
        Compute RAD score with this ratio
        Compute Spearman ρ vs FP32 score
    record min ρ across layers

Pick smallest ratio where min ρ ≥ 0.90
```

**Safety margin**: 取最小可行 ratio. 跨 model 觀察 37.5% 為 sweet spot (n=2 待 confirm).

### 2.4 (n_Q, n_K) anchor 組合

**方法**: 完整 sweep on hardware-aligned combos

**約束**: `n_Q + n_K ∈ {1, 2, 4, 8}` (硬體 routing 對齊)

**演算法**:
```
For each (n_Q, n_K) with n_Q + n_K ∈ {1, 2, 4, 8}:
    Compute worst-case Spearman ρ across representative layers
    
Pick combo with highest worst-case ρ
```

**Observation**: 不同 model best combo 不同 (e.g., TinyLlama 1Q+1K, Qwen 1Q+3K). **必須 per-model calibrate**.

### 2.5 K-anchor table per (layer, head)

**方法**: Calibration set K-magnitude maximum

```
For each (L, head_kv, pair_idx):
    Track max |K| over calibration set
For each (L, head_kv):
    Pick top-n_K pairs (excluding Q-anchor indices) → K-anchor index
Write to BRAM table
```

### 2.6 Q-anchor index per (layer, head)

**方法**: Calibration set Q-magnitude global average

```
For each (L, head_q):
    Compute global mean(|Q_pair|) across calibration tokens
    Pick top-n_Q pairs → Q-anchor index
Write to BRAM table
```

### 2.7 mean(|K_drop|) per (layer, head)

**方法**: Calibration set expected value

```
For each (L, head_kv):
    Compute mean over (calibration tokens) × (dropped dim of |K|)
Write to BRAM table (16-bit fixed-point)
```

### 2.8 block_size `B`

**方法**: Chebyshev statistical bound + dispatch overhead trade-off

**Default**: `B = 16` (Chebyshev P(error) < 5%, dispatch rate ~30%)

### 2.9 T_margin, T_overflow (dispatch thresholds)

**方法**: Chebyshev confidence interval

```
T_margin = 0.75   → 3σ, P(error) ≤ 11%
T_overflow = 0.25 → 1σ, cover 70% candidates
```

## 3. Calibration Flow (跨 model deployment, ~10 min)

1. Load target model + tokenizer
2. **Run 5 sample forward**, hook K_proj / Q_proj output
3. **Compute K-anchor table**: per-(L, head, pair) max |K|
4. **Active dim sweep**: Spearman ρ on 4-5 representative layers, exclude L0
5. **Anchor combo sweep**: completing n_Q + n_K ∈ {1,2,4,8}, worst-case ρ standard
6. **β sweep** (200 sample): verify β=1.0 optimal (theorem confirm)
7. **Write BRAM tables**: K-anchor, Q-anchor, mean(|K_drop|)
8. **Write registers**: active_lane_count, n_Q, n_K, β
9. **KIVI G**: 用 default G=32, 不需 calibrate

**Note**: KIVI-V 沒有 per-model tunable threshold, calibration flow 比 PxV 簡化.

## 4. 硬體實作 (parameterized RTL)

### 4.1 RTL 參數化常數

```systemverilog
// Compile-time parameters (model-portable)
parameter D_MAX        = 128;  // Max head_dim (LLaMA2 7B head_dim=128)
parameter P_MAX        = 64;   // Max num_pairs = D_MAX / 2
parameter L_MAX        = 32;   // Max num layers (LLaMA2 7B 32 layer)
parameter HQ_MAX       = 32;   // Max Q heads
parameter HKV_MAX      = 32;   // Max KV heads
parameter B_OPTIONS    = {8, 16, 32};  // Block size options
parameter ACTIVE_OPTIONS = {0.125, 0.1875, 0.25, 0.3125, 0.375, 0.5, 0.625};
parameter ANCHOR_TOTAL = {1, 2, 4, 8};
parameter LANE_COUNT   = 6;    // PE Lanes (固定)
parameter PE_PER_LANE  = 8;    // PE per lane (固定)
parameter KIVI_GROUP_SIZE = 32; // V quantization group size
```

### 4.2 Runtime 配置 register

```
+----------------------------+
| reg_head_dim          [7:0]   // 64 / 80 / 96 / 128
| reg_num_layers        [5:0]
| reg_num_q_heads       [5:0]
| reg_num_kv_heads      [5:0]
| reg_active_lane_count [3:0]   // 1 ~ 6
| reg_n_Q_anchor        [2:0]   // 0 ~ 4
| reg_n_K_anchor        [2:0]   // 0 ~ 4
| reg_block_size        [5:0]   // 8 / 16 / 32
| reg_BETA              [3:0]   // β shift amount (β=1.0 → shift 0)
| reg_T_margin          [7:0]   // margin threshold × 256
| reg_T_overflow        [7:0]   // overflow threshold × 256
| reg_M_local           [2:0]   // 1 ~ 4
+----------------------------+
```

註: 沒有 P_THRES (取代 PxV 後不需要).

### 4.3 BRAM tables (per-deployment)

| Table | Size | Per (L, head_kv) entries |
|-------|------|--------------------------|
| K_anchor_idx | `L × HKV × n_K × ceil(log2(P_MAX))` bit | n_K indices |
| Q_anchor_idx | `L × HQ × n_Q × ceil(log2(P_MAX))` bit | n_Q indices |
| mean_K_drop | `L × HKV × 16` bit | 16-bit fixed-point |

### 4.4 整體架構

```
KV260 FPGA
├── QK-PU
│   ├── PE Lane × LANE_COUNT (each PE_PER_LANE PE)
│   ├── Cross-bar dim selector (per-lane, cascading 4:1 mux)
│   ├── Block-Sum-Indexing scheduler (BS)
│   └── signal_amp / margin runtime compute
├── Softmax
│   ├── Base-2 softmax (replace e^x with 2^x for LUT-based)
│   └── Reciprocal LUT (replace divide with multiply)
├── V-PU (KIVI-V)
│   ├── 16×16 systolic array (output-stationary, pure 4-bit MAC)
│   ├── Per-group scale-extraction multiplier (1 per group output)
│   └── Per-group min offset adder
├── Memory
│   ├── K Buffer (BRAM, K-stationary, 8-bit)
│   ├── V Buffer (BRAM, V-stationary, 4-bit data + INT8 scale + INT8 min)
│   ├── Q Buffer (BRAM, 8-bit)
│   └── Anchor + mean(|K_drop|) tables (BRAM)
└── DDR4 (weights + KV overflow for S > BRAM capacity)
```

### 4.5 PE Lane 設計 (parameterized)

每 lane: `PE_PER_LANE` PE
- Active path: PE_PER_LANE × 8-bit MAC + tree adder → (11 + log2(PE_PER_LANE))-bit partial sum
- SIGN path: PE_PER_LANE × XNOR-AND + |Q| accumulate → 同 width
- Time-multiplexed: active 跟 SIGN 共用 PE (序列執行)

跨 lane reduction tree:
- `LANE_COUNT × (11 + log2(PE_PER_LANE))`-bit input → 14-bit final
- Pipeline 2 stage, each ≤ 11-bit carry chain
- Target 200 MHz

### 4.6 Cross-bar dim selector

- `LANE_COUNT × PE_PER_LANE` cascading 4:1 mux
- LUT cost ~1K
- Single-cycle dim selection per query

### 4.7 Softmax 硬體化

**Base-2 softmax**:
```
score → s
2^s = 2^(int) × 2^(frac), 2^(frac) 用 LUT
Sum = Σ 2^s
P_i = 2^s_i × (1 / Sum), 1/Sum 用 reciprocal LUT
```

註: 取代 PxV 後不需 `2^s > P_THRES × Sum` 比較 logic, Softmax 直接接 V-PU.

### 4.8 KIVI-V 硬體實作

**Per-group MAC**:
```
For each output element o:
    For each (P_token, V_int4_token, scale_token, min_token):
        For each group g of G consecutive V channels:
            For each channel c in group:
                # INT4 V × INT8 P, partial sum in group
                partial_sum_g += V_int4[c] × P[token]
            # group 結束才 dequant
            scaled_g = partial_sum_g × scale_g + offset_g × Σ(P_token in this group)
        accumulate across groups
    o = sum over all tokens
```

**Scale-extraction trick**: scale 提到 group 累加完成後乘 1 次, 而不是每個 MAC 都 dequant. 省下 G-1 個 dequant 乘法器.

**Memory layout (separated)**:
```
V_data buffer:    [V0_INT4, V1_INT4, ..., V_{G-1}_INT4]  per group, 128-bit aligned
V_scale buffer:   [scale_g, min_g] (INT8 + INT8 = 16-bit) per group, separate BRAM
```

**對齊**: V_data 跟 V_scale 在不同 BRAM bank, 並行讀取無對齊問題.

| `num_kv_heads` HKV | 1 ~ HKV_MAX | runtime register |
| `active_dim` A | 0.125 D ~ 0.625 D | active_lane_count register |
| `n_Q` | 0 ~ 4 | Q-anchor BRAM 控制 |
| `n_K` | 0 ~ 4 | K-anchor BRAM 控制 |

| `T_margin` | 8-bit | register |
| `T_overflow` | 8-bit | register |
| `KIVI_group_size` | 32 (固定) | compile-time |

### 4.10 Fusion

**Fusion 1**: Active + SIGN compensation in PE lane (time-multiplexed)
**Fusion 2**: QK + Softmax + KIVI-V pipeline (FlashAttention-style tiling, tile=128)
**Fusion 3**: Scale-extraction in V-PU (group-level dequantize)

### 4.11 Layer 0 異質管線 mapping

硬體有 `LANE_COUNT × PE_PER_LANE` 物理 PE (default 48).

Layer 0 跑 `ceil(D / (LANE_COUNT × PE_PER_LANE))` iteration:
- 對 D=64, LANE×PE=48: 2 iteration (48 + 16, 4 lane idle 在 iter2)
- 對 D=128, LANE×PE=48: 3 iteration

Layer 0 throughput slower 2-3x, 但 only 1/L of layers, 整體影響 < 10%

### 4.12 SDH (Software-Defined Hardware) 範圍

跨 model 跨 task reconfigurability, **不是 per-layer adaptive**:
- RTL parameterize 提供範圍
- Calibration table per model (K/Q-anchor index, mean|K|)
- Runtime register 配置 (active_lane, n_anchor, β)

### 4.13 資源估算 (KV260 target)

| 模組 | LUT | DSP | BRAM |
|------|-----|-----|------|
| QK-PU (LANE × PE) | ~5K | LANE×PE | 0 |
| Cross-bar dim selector | ~1K | 0 | 0 |
| Softmax (base-2 + recip LUT) | ~1.5K | 4 | 0 |
| V-PU (8×16 systolic, 4-bit) | ~3K | 96 | 0 |
| Per-group scale extract | ~0.7K | 16 | 0 |
| BS scheduler | ~1K | 0 | 0 |
| Control + FSM | ~3K | 0 | 0 |
| K Buffer (8-bit) | 0 | 0 | 256 KB |
| V data Buffer (4-bit) | 0 | 0 | 128 KB |
| V scale Buffer (INT8+INT8) | 0 | 0 | 32 KB |
| Q Buffer (8-bit) | 0 | 0 | 32 KB |

**Bottleneck**: BRAM 70%, DDR4 bandwidth, **not** compute throughput

### 4.14 Memory wall 應對 (orthogonal optimization)

| 技術 | 收益 | 與 RAD+KIVI 相容 |
|------|------|-----------------|
| W4A8 weight quantization (AWQ/GPTQ) | DDR4 BW 省 50% | ✓ orthogonal, AWQ+RAD+PxV 已驗證 (drop 2.18pp), KIVI 預期類似 |
| Group-wise KV cache compression | 已內建 (KIVI is KV cache INT4) | ✓ KIVI 本身就是 KV cache 壓縮 |
| Hardware batching | DSP util ↑ | ✗ edge 場景不適 |

## 5. Simulation Caveats (model-agnostic)

### 5.1 GPU simulation 範圍

當前 GPU simulation 涵蓋:
- ✓ 機制邏輯 (dim selection, SIGN compensation, block dispatch, KIVI quantization)
- ✓ V 4-bit group-wise RTN quantization (explicit 模擬)
- ✗ INT8 active MAC quantization (未模擬, GPU 跑 FP16/FP32)
- ✗ INT8 累加溢位 (未模擬)
- ✗ Softmax 8-bit fixed-point (未模擬)

### 5.2 MAC counter 定義

- `total_baseline_macs`: full FP MAC count
- `total_stage1_macs`: active dim × MAC count
- `total_stage2_macs`: candidate token 補算 MAC
- `total_v_rad_macs`: `total_valid × D × 0.5` (4-bit-equivalent, 全 4-bit)

**注意**: `× 0.5` 是「位寬等效 cost」, 不是 MAC operation 數量.

### 5.3 待補實驗

1. INT8 active MAC fake quantization 驗證 (~30 min)
2. AWQ + RAD + KIVI-V 完整疊加 (~75 min)
3. Cross-task on KIVI (ARC-Easy, ARC-Challenge, PIQA, Winogrande, ~2 hr)
4. Qwen2.5-1.5B + RAD + KIVI-V (~75 min)
5. PG-19 long-context on KIVI (~30 min, 預期優於 PxV)
