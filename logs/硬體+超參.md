# Transformer Attention + FFN Co-Sparse Accelerator — 機制與硬體規格 (v6)

**Target**: Xilinx KV260 FPGA (117K LUT, 1248 DSP, 4GB DDR4 64-bit @2400 ≈ 19.2 GB/s peak; 片上 SRAM = BRAM 5.1 Mb + URAM 18 Mb ≈ 23 Mb, 兩種資源分開預算, 不可混稱 BRAM)
**Scope**: LLaMA-family decoder-only Transformer，GQA + SwiGLU + RoPE。**主 model: Llama-3.2-3B-AWQ** (D=128, GQA 3:1, ctx 128K); TinyLlama / Qwen2.5 為 cross-model 驗證
**前提**: AWQ INT4 weight-only 量化模型 (premise, 非可選項; 硬體 datapath 依賴之, 不支援非 AWQ 路徑)

---

## 0. 系統總覽

兩個正交稀疏機制 + 全棧量化：

| 區塊 | 機制 | selection 粒度 | 省的資源 |
|------|------|---------------|---------|
| Attention (Q·K·V) | RAD-Cascade + KIVI-V | token-level (active dim + Global Top-N) | Q·K MAC + KV cache BW |
| FFN (SwiGLU) | CETT activation sparsity | neuron-level (真實貢獻 ‖n_i‖ 閾值) | down MAC + down 權重 BW |

兩機制統計獨立 (整合實測交互項 ≈ +0.09 pp，近線性疊加)，共用 Sparse Gather Engine。

memory-bound 平台 (KV260 單通道 DDR4)：**BW 省 > MAC 省**，設計優先壓 KV cache BW 與權重 BW。

## 0.1 全棧位寬與資料流 (v6.1 定案, 全部實測, 位寬階梯見 2.11)

原則: **MAC 向量路徑一律整數/精確定點; FP16 只出現在「攤提後標量」或「元素級 O(d) 操作」**, 每處附 0.2 實作。

| 資料 / 運算 | 位寬 | 實測 Δ | 說明 |
|------|------|--------|------|
| 權重 (所有 linear) | **INT4** (AWQ) + per-group scale (FP16 標量, group=128) | premise | DDR 存 INT4; **不做逐元素 dequant** — scale 是 group psum 後標量乘 (1/128 攤提) |
| linear MAC | **FP16-act mantissa-shift × INT4** (Kulisch, 0.2①) | **精確** (0 量化誤差, 僅 exp window clamp) | `(m_x × W_INT4) << e_x` → 48-bit 定點累加; per-group psum × s_g 標量。**A8 已驗死** (見 dead directions: qkv +0.35% 與 RAD 選擇交互; 混合配置零硬體收益被支配) |
| 激活 hidden_states | **FP16** | — | AWQ weight-only 前提維持; MAC 成本已由 mantissa-shift 解掉, 不需 A8 |
| residual + RMSNorm | FP16 (元素級 O(d)) | — | rsqrt LUT+NR (0.2③); 加法器時分 (0.2④) |
| RoPE | **定點 INT16 sin/cos LUT** (Q1.15) | **-0.01%** | 旋轉後量化 INT8 |
| Q·K (stage1/2) | **INT8 × INT8** | (v6 既有) | psum ≤23-bit; SIGN sign-XOR 同樹 |
| K-anchor running-max | FP16 寄存器 | — | 只比大小 (FP16 正數比較=整數比較, 零 FP 邏輯) |
| KV cache: K | **INT8**; dim-major 副本 + sign plane = long-context 配置 (4.5.2) | (v6 既有) | |
| KV cache: V | **INT4** (KIVI G=32) | (v6 既有) | |
| Softmax | **全定點**: 整數 max-sub → 輸入 1/256 (exp2 LUT 8-bit fraction) → reciprocal LUT | **+0.02%** (含 U16) | 無 FP |
| attn_w | **U16 定點 Q0.16**, 不重歸一化 | (併入上) | softmax 輸出 ∈[0,1], 全網唯一有界張量, FP16 exponent 浪費 |
| attn_w × V | **U16 × U4 整數** + per-token-group 係數摺疊 | 結合律重排, 0 額外誤差 | `out_d = Σ_t (w_t·s_{t,g})·Vq + Σ_t w_t·m_{t,g}`; c 標量 1/32 攤提; **不解壓 V** |

- 位寬全套合計 (RAD shared-dim + RoPE16 + 定點 softmax/U16 + V 整數 MAC): **PG-19 12.3616, vs AWQ dense +1.18%** (nc=50, S=2048)。
- **權重 BW 精算**: INT4 + group=128 FP16 scale = 4.125 bit/weight → 省 74.2%。

## 0.2 FP16 必留處與硬體實作 (LUT/DSP)

**① 統一 GEMV lane — mantissa-shift 寬累加 (Kulisch 式)**
```
MAC_i = (m_x × W_INT4) << e_x   → 48-bit 定點累加器
  m_x = FP16 mantissa (11-bit, 含 hidden bit), e_x = exponent
```
- **精確運算**: mantissa×INT4 整數積移位後寬累加無捨入, 比 FP16 FMA 鏈更準; 唯一近似 = exponent window clamp
- 成本/lane: 乘法 11×4-bit → 27×18 DSP **可 2-pack** (~0.5 DSP) + barrel shifter ~30 LUT + 48-bit 累加 (DSP48E2 ALU 恰 48-bit, 或 fabric)
- exponent window [2⁻¹², 2¹³]: 離線校準激活 exponent 直方圖定窗 (3.1 流程), **飽和計數器** runtime 監控
- V mode: 同一 27×18 乘法器吃 U16(attn_w) × U4(Vq), shift=group 係數對齊 — linear / attn·V 一套硬體

**② scale 標量乘 (per-group s_g, per-token a_s, KIVI c_{t,g})** — 同 ① trick: `psum(INT) × m_s << e_s`, 攤提 1/128 (權重) / 1/32 (V), 共 ~4-8 個此類乘法器全晶片。

**③ rsqrt (RMSNorm)**: LUT 2⁸×16b + 1 次 Newton 迭代, 1 DSP, per-token per-norm 1 次 (O(1))。

**④ residual FP16 加法器**: ~70 LUT/個 × 8 個時分 (per token d 次加法, ~10 tok/s 下吞吐裕度 >100×)。

**⑤ exp2 / reciprocal LUT (softmax)**: BRAM 2⁸×16b 各一, 0 DSP。

## 1. 機制 (model-agnostic)

### Notation

| Symbol | 定義 |
|--------|------|
| `L` | 層數 |
| `H_Q` / `H_KV` | Query / KV head 數 (GQA) |
| `D` | head_dim |
| `P` | num_pairs = D / 2 |
| `S` | sequence length |
| `A` | active pairs per query |
| `n_K` | K-anchor pair 數 |
| `N` | Global Top-N candidate 數 |
| `G` | KIVI V group size |
| `d_ff` | FFN intermediate size |
| `τ_CETT` | FFN CETT 閾值 (PPL+1% 標準對應) |
| `ε_i` | 第 i 層 FFN neuron 貢獻閾值 (τ_CETT 反推, 離線校準) |

### Layer-wise dispatch

| Layer | Attention | FFN |
|-------|-----------|-----|
| Layer 0 | 異質: 跑滿 D dim (active=P, n_K=0), INT8 Q·K + INT4 V, 不套 RAD selection | CETT activation sparsity |
| Layer 1 ~ L-1 | RAD-Cascade (active + SIGN + Global Top-N) + KIVI-V | CETT activation sparsity |

Layer 0 異質原因: per-row attention 分散度高 (max ≈ 1/S), RAD selection 不適用; 跑滿維度, 後續層才省。

### 1.A Attention: RAD-Cascade

量化: Q, K per-token INT8 (進 PE 後轉)。

#### Stage 1: Active path + SIGN 補償 (Q·K 端)

**Active dim 選擇** (INT8 MAC, full-D-dim einsum with mask):
- 每 query 選 `A` 個 active pair，兩類組成：
  - `n_K` 個 **K-anchor pair**: per-position streaming running-max of |K| (FP16) 選 top-n_K (見 2.7)
  - `A - n_K` 個 **dynamic pair**: **GQA shared-dim** — per KV-head group (共用同一 K 的 G_Q = H_Q/H_KV 個 Q head) 以 `Σ_h |Q_h|` pair-mag 選 top, anchor scatter 排除; 組內所有 Q head 用同一組 dim。**選擇輸入用量化前 FP16 幅度** (v6.1: selection 是純比較, FP16 比大小=整數比較, 零成本; A8 分解證明選擇對 Q/K 擾動敏感 → 對症, 實測 -0.03%), MAC 仍 INT8
- **為何 shared 而非 per-head (GQA union 效應, 實測)**: K 每步 per KV head 只讀一次, per-head 各選 24 dim 時實際讀取 = 8 head 的 dim union = 82.0% ± 7.0% (L10 實測; 獨立上界 97.7%) → K BW 省從單 head 62.5% 塌到 18%。shared-dim 把 union 收斂回 24 dim (省 62.5% dim-level), 端到端代價 **+0.14% PPL** (PG-19 nc=50, vs per-head, noise floor 內)。MAC 省不受 union 影響 (per-head 本來就只乘自己的 dim), shared-dim 是純 BW 修正。
- pair → dim: `full_dim_mask = pair_mask.repeat_interleave(2)`
- `active_score = einsum(Q_b ⊙ full_dim_mask, K_b)` (mask 乘法後全 D 維 einsum, 等價只算 active 維)
- 不使用 Q-anchor (n_Q = 0, 見 2.5)

**SIGN 補償** (純 sign, 1-bit sign(K) + 8-bit |Q| 累加, 零乘法器):
```
sign_k = (K_b >= 0) ? +1 : -1
comp   = einsum(Q_b ⊙ dropped_mask, sign_k)    # Σ_d∈dropped  q_d · sign(k_d)
stage1 = active_score + β × comp
```
- β = 1.0 (Theorem 1)
- **純 sign-only, 無 mean(|K_drop|) 縮放** (sign-XOR 直接進 adder tree, 不需 |K| 統計表)

**Theorem 1** (SIGN unbiased): 對稱分布下 E[sign(k)·k] = mean(|k|), β=1.0 為 unbiased estimator。

#### Stage 2: Global Top-N selection (prefill soft-tail / generation hard-truncation)

```
full_score    = einsum(Q_b, K_b)                       # 全 D 維 INT8 (候選用)
stage1_masked = where(valid_causal, stage1, -inf)
topN_idx      = topk(stage1_masked, N)                  # row-global top-N
cand_mask     = scatter(topN_idx) & valid_causal
# prefill : final_score = where(cand_mask, full_score, stage1)   [soft-tail, 讀全 V]
# generation: final_score = where(cand_mask, full_score, -inf)   [hard-trunc, V 讀 per-KV-head union]
```

**prefill 與 generation 分開處理 V (關鍵, 實測決定):**

| 階段 | 非候選 score | V 讀取 | 理由 |
|------|-------------|--------|------|
| **prefill** | 保留 stage1 近似 (soft-tail) | 讀全部 V | hard-trunc 崩 (見下) |
| **generation** | -inf (hard-truncation) | 只讀候選 V (per-KV-head **union**) | 可行, 省 V BW **~91% @N=64** (GQA 真值, 見下) |

- **prefill 必須 soft-tail (實測證據)**: prefill 全 query 同時 hard-truncation (非候選 -inf, 只讀 top_n V) → PG-19 PPL **+183% 崩潰** (S 個 query 每個表徵砍到只剩 N target, 累積毀掉所有位置)。prefill 保留 stage1 近似才不崩。
- **generation 可 hard-truncation (實測證據)**: generation 每步單 query 對歷史 KV 選 top_n, 非候選 -inf 只讀候選 V → XSum ROUGE-1 drop 僅 0.91 (vs soft-tail 0.58, 多掉 0.33), **不崩**。同 SparQ Step2 (decode 階段只 fetch top-k V), prefill 不做。
- **GQA 真值 (top-N 維持 per-Q-head, 讀取按 union)**: V per KV head 只讀一次, 實際讀取 = G_Q 個 Q head 候選集的 union。實測 (L10, S=2048): union = 77.5±10.4 tok @N=32 / **137.8 tok @N=64 (採用)** → V BW 省 **92% / ~91%** (保守口徑 union/平均causal長度; union/S 口徑 96.6%/93.3%), 非 per-head 帳面的 98%。
- **shared top-N 已驗死** (見 dead directions): 強制 8 head 共用 N_g=32 候選 → PPL +4.02% (vs shared-dim only); N_g=64 僅比 union 77 省 17% 還丟 per-head 資訊, 無存在價值。**候選選擇保持 per-head, BW 帳按 union 算**。
- **與 SparQ 異同**: top-k token 選擇 + dim 近似框架同 SparQ (公開方法, 已引用); 增量為 SIGN 補償 (stage1)、K-anchor running-max、**GQA shared-dim selection (per-head 選擇在 GQA 下 BW 省塌縮, SparQ 未處理; 待查原文確認)**、INT8/INT4 量化整合、FPGA MAC 省。
- `N = f(S)` (見 1.C)；row-global，非 per-block 配額。

#### Stage 3: KIVI-V + attn_w

- attn_w (softmax 輸出): **U16 定點 Q0.16** (定點 softmax, 實測 +0.02%, 見 2.11)
- V: per-token group-wise INT4 (KIVI, Liu et al. 2024), default `G = 32`
- **attn_w × V**: U16 × U4 整數 MAC + per-token-group 係數摺疊 (不解壓 V, 見 0.1)
  - **prefill**: 讀全部 V (soft-tail 需全 V 參與)
  - **generation**: 只讀 top_n 候選 V (非候選 softmax=0)
```
For each group g of G channels:
    scale_g = (max-min)/15;  V_q = round((V-min)/scale_g) ∈ [0,15]
attn_out = attn_w(FP16) @ V_dequant(FP16)
```
- **V BW 省**: prefill 靠 INT4 壓縮 (~75%); generation 額外靠候選截斷 (讀 per-KV-head union ≈ 138/2048 @N=64, 省 **~91%**)。

### 1.B FFN: CETT Activation Sparsity

#### 原理
SwiGLU: `out = W_down( SiLU(x·W_gate) ⊙ (x·W_up) )`。每 neuron 對 MLP 輸出的真實貢獻向量 `n_i = h_i · W_down[:,i]` (h_i = SiLU(gate_i)·up_i)。跳過低貢獻 neuron → 其 W_down[:,i] 不算/不載 → 省 down MAC + down 權重 BW。

#### Selection: CETT ε 閾值 (training-free, 動態)
判據用 neuron **真實貢獻範數** (含 down 投影, 比純 gate proxy 準):
```
‖n_i‖ ≈ |h_i| × ‖W_down[:,i]‖           # h_i 標量 × down 列範數 (SRAM 常數)
跳過 ‖n_i‖ < ε_i 的 neuron               # 動態, 砍量隨輸入/token 變
```

**CETT 度量** (TDA Eq.20, 校準 ε 用):
```
D = { i | ‖n_i‖ < ε }
CETT = ‖ Σ_{i∈D} n_i ‖₂ / ‖ MLP_output ‖₂     # 被砍 neuron 輸出「向量和」範數 / 完整輸出範數
```
- 分子為**向量和的範數** (有方向抵消)，非標量平方和。TinyLlama 上向量抵消可忽略 (與標量近似差 <2%)，但定義以 TDA Eq.20 為準。

#### 標準: PPL 漲 1% (文獻泛化, 非 overfit 固定比例)
- τ_CETT = 使 validation PPL 漲 ~1% 的 CETT 值 (Sparsing Law arXiv 2411.02335 證明此標準泛化性最強、高稀疏退化最小)。
- 每層 ε_i 由 τ_CETT 反推 (二分搜，離線校準，~8 chunk 收斂)。
- **τ_CETT 是 model-agnostic 標準** (「損失 1% PPL」任何模型同義); sparsity 是結果 (隨 model/輸入自適應)，不寫死。

#### prefill 與 generation 統一動態
- prefill 與 generation **都用 CETT ε 動態砍** (每 token 算 ‖n_i‖ 比 ε_i)，砍量隨輸入變。
- **不使用固定 mask 複用** (TDA-style prefill 定 mask、generation 重用): 實測 generation 固定 mask 與動態需求重合度僅 69.9% (淺層 L0=43.6%), 砍 43% → ROUGE 崩 5.26; 換 Llama-3.2-3B 重合度 68.6% (非規模問題, 固定 mask 本質不適用)。詳見 dead directions。

#### 機制邊界 (為何只省 down)
training-free + 不改激活函數 + SwiGLU 三約束下，**只能省 down**:
- 要選哪些 neuron 砍，須先算 h_i = SiLU(gate)·up → **gate/up 必須全算** (mantissa-shift lane, 不省)。
- SwiGLU (SiLU) 無硬零 (不像 ReLU)，激活不可廉價預測 → 無法免訓練 predictor 預測 mask 來省 gate/up。
- 文獻中省 gate/up 者皆需 (a) train predictor (DejaVu/PowerInfer) 或 (b) 改激活函數為 ReLU²/FATReLU 並續訓 (ProSparse 80%) → 違反 training-free + no-modification。
- TDA 論文亦明言「only down projection reduced <50%, L2 norm 仍需算」→ 只省 down 是約束的數學本質，非實作不足。

#### CETT vs 舊 gate-proxy (為何升級)
- gate proxy (|SiLU(gate)|×‖W_up‖) 上限 ~20% (gate⊥up 統計獨立, corr≈0)，砍 25% 即 ROUGE 崩。
- CETT (含 down 投影 ‖n_i‖) + ε 動態 + PPL+1% 標準 → 無損砍 ~43% (TinyLlama PG-19 PPL +0.94%, XSum ROUGE-1 drop 0.70)，**翻倍且更嚴謹**。

### 1.C 序列級自適應 (active 與 top_n 隨 S 配置)

attention 兩稀疏參數依**序列長度 S** 在 prefill (S 已知) 配置一次，**整段固定** (非 per-query → MAC 陣列配置一次, 無 pipeline 氣泡)。roofline: 短序列 compute-bound (省 MAC), 長序列 memory-bound (精度優先, BW 靠 top_n + KV 量化省)。

**active: 固定 37.5% (序列級雙檔已刪, v6.1)**:
- 舊設計「短序列降 25% 省 MAC」與數據矛盾: S=512 sweep 全配置不過 PPL+1% (37.5%/32 已 +2.66%), 短序列精度餘裕**更小**; 且短序列 Q·K MAC 占比僅 0.24%, 無省的價值。刪雙檔, 固定 37.5%。
- active dim 仍須 DMA 對齊 (24 dim @ D=64); per-model 校準只定比例 (25/37.5/50% 三 candidate)。

**top_n=f(S)** (四檔, v6.1 長 S 加 64 檔):
| S | top_n |
|---|-------|
| S < 128 | 8 |
| 128 ≤ S < 256 | 16 |
| 256 ≤ S < 1024 | 32 |
| S ≥ 1024 | **64** |

- 舊設計封頂 32 與 S 脫鉤 (S=2048 候選覆蓋 1.6%): N sweep @ S=2048 (見 2.12) 證明 N=64 → PPL -0.43%, **attention 全棧壓進 +1% 標準** (vs AWQ +0.74%); 邊際遞減 (32→48 -0.25pp, 48→64 -0.15pp) 支持停在 64。V union 138 → V 省仍 >91% (保守口徑), BW 近零代價。S=1024 切點為內插, 必要時 per-S 補驗。

- top_n 影響 Stage 2 gather 候選數 (gather 層面可變, 不破壞 MAC 陣列)。短序列 Stage 2 占 MAC 比例高 (省 top_n 省大量 MAC); 長序列占比小, top_n=32 維持精度。

**為何序列級固定而非 per-query / per-layer**:
- per-query 動態 active → MAC 陣列每 query 重配 → 氣泡。序列級配置一次, 無氣泡。
- per-layer 差異化 active 無 model-agnostic 免校準判據, 不採用。
- top_n 動態為 gather 層面, 不觸 MAC 陣列, 故隨 S 配置無氣泡。

### 完整 dataflow
```
Per layer:
  RMSNorm (FP16, rsqrt LUT+NR)
  [Attention]
    QKV 投影: x(FP16) @ W_qkv(INT4): mantissa-shift lane, per-group psum × s_g → Q/K/V
    RoPE (INT16 sin/cos LUT) → Q,K → 量化 INT8
    K: |·| 更新 running-max 錨點 (片上); INT8 寫 cache (row-major; dim-major 副本依 long-context reg)
    V: 量化 INT4 寫 cache
    Per query (generation: per step):
        1. K-anchor: top-n_K of 當下 K_runmax
        2. Dynamic : per KV-head group Σ_h|Q_h| pair-mag 選 top-(A-n_K) (shared-dim, anchor 排除)
        3. Stage 1 : active_score (INT8) + β·SIGN (sign-XOR, 零乘法)
        4. Stage 2 : full_score(INT8) on per-head top-N candidate
                     prefill: final = where(cand, full, stage1)  [soft-tail, 讀全 V]
                     generation: final = where(cand, full, -inf)
                     generation gather: 8-head 候選 union → 單一 index stream 同時取 K row + V row
        5. Softmax (全定點: max-sub → exp2 LUT → reciprocal LUT) → attn_w (U16 Q0.16)
        6. attn_w(U16) × Vq(U4) 整數 MAC + per-token-group 係數摺疊 → attn_out (不解壓 V)
    o_proj: attn_out @ W_o (mantissa-shift lane); residual add (FP16)
  [FFN]
    RMSNorm (FP16)
        7. gate = x·W_gate, up = x·W_up   (mantissa-shift lane, 全算)
        8. h_i = SiLU(gate_i)·up_i (SiLU LUT)
        9. ‖n_i‖ = |h_i| × ‖W_down[:,i]‖(SRAM); keep = ‖n_i‖ ≥ ε_i  (動態)
       10. down: gather keep 的 W_down column → ffn_out; residual add (FP16)
```

```
│   └── 服務 attn_w·V (KIVI Dequant 餵 INT4→FP16 V row) — 同一 scalar×row 累加 dataflow, 分時
├── Unified Top-K Engine (≤64)          ── K-anchor / Dynamic / Global Top-N 分時共用
├── Softmax (base-2 + reciprocal LUT) → attn_w FP16
├── Sparse Gather Engine                ── attn Stage2 K+V (單 index stream) / FFN down neuron 共用 DMA

├── Weight Dequant Unit                 ── INT4 → FP16 (per-group scale), 餵 GEMV Engine
└── Memory (K/V/Q Buffer [URAM]; K_runmax + 小表 [BRAM]; K 雙 layout + sign plane + W_down_norm + AWQ weights [DDR4])
```

### 4.5 硬體資源共用

**① 統一 Top-K Engine (≤64) — 三 stage 共用**
K-anchor (k≤4)、Dynamic pair (k≈A-n_K)、Global Top-N (k=N≤64) 皆「分數流選 top-k」，k≤64 且時間不重疊。共用 64-entry sorted shift register，k 由 register 配置。省 2 套選擇器。

**② FFN selection 不併入 Top-K Engine — 獨立 CETT Filter**
FFN 留 neuron 數巨大 (千級)，streaming 全排序面積爆炸。改用 **CETT 閾值過濾** (‖n_i‖ vs ε_i, 過線即留)，非全排序。獨立成 CETT Filter Unit。

**③ Sparse Gather Engine — attn Stage 2 與 FFN down 共用**
Stage 2 token gather 與 FFN down neuron 權重 gather，「地址生成 → read request → 收亂序回傳」狀態機相同。共用同一 DMA gather 控制器。解決 memory wall 核心整合。

**④ Adder Tree — Active MAC 與 SIGN 共用**
SIGN = Σ|Q_d|·sign(K_d): sign(K) 對 Q 做 XOR 取反 (不需乘法器), 進 active path 同一 Adder Tree。active 與 SIGN time-multiplex 共用。

**⑤ Weight Dequant Unit — 所有 linear 共用**
INT4 權重 + per-group FP16 scale → FP16, 供投影/o_proj/gate/up/down。register-level (送進 MAC 前一刻 dequant), 不增 DDR BW。

**⑥ Linear/V GEMV Engine — 所有 linear 與 attn_w·V 共用 (v6 修正, 取代獨立 V-PU)**
generation (batch=1) 下所有 linear 都是 GEMV: `x@W = Σ_i x_i × W[i,:]` (scalar × row 累加); 而 `attn_out = Σ_t attn_w_t × V[t,:]` 是**同一 dataflow**。一套 FP16 vector-accumulate 引擎 (GEMV_LANES=128) 分時服務兩者, 差異只在 row 來源 (Weight Dequant vs KIVI Dequant)。砍掉舊獨立 V-PU 16×16 systolic, **省 ~96 DSP + 一套控制 FSM**。generation BW-bound, MAC 大量閒置, 共用零吞吐代價。
舊 spec「QK-PU 與 V-PU 不共用」論證只對 QK 成立 (scalar-reduction vs vector-accumulate 正交), 對 V-PU vs linear 不成立 (兩者同為 vector-accumulate) — 且舊資源表**完全沒給 linear (占總 MAC 73-78%) 任何運算單元**, 此為 v6 前最大漏洞。

**不共用: QK-PU 與 GEMV Engine** — reduction 方向正交 (QK 歸約成 scalar 用 adder tree; GEMV 跨 row 累加成 vector), 且位寬不同 (INT8 vs FP16)。保持兩套獨立 datapath。

### 4.5.1 序列級自適應配置單元 (active/top_n = f(S))
```
At prefill (S known):
    top_n = (S<128)?8 : (S<256)?16 : (S<1024)?32 : 64   // Unified Top-K Engine 的 k (active 固定 37.5%)
```
純比較器, ~0 額外 DSP, 序列開始配置一次 (無氣泡)。DMA 對齊保證。

### 4.5.2 K cache 雙 layout (DDR burst 對齊, K BW 省成立的前提)

**問題**: D=64 INT8 → 一個 K row = 64B = 恰好一個 DDR4 burst。row-major 單份存放下, 「只讀 active dim」仍觸發整 burst → **dim-skip BW 省 = 0**, §5.2 的 K stage1 省宣稱不成立。

**自攻過的替代 (皆死)**:
- dim-major 單份: stage2 gather (40 dropped dim × 候選 token 散讀) ≈ 80KB, S=2048 時吃掉 stage1 省的 71.7KB, 淨省歸零
- block-transpose tile (tile 內 dim-major): dim 讀滿 burst 需 tile ≥64 token → stage2 一個候選讀整個 4KB tile, 慘 64 倍

**採用: 雙 layout (SparQ 原文同做法, store K twice)**:
| 份 | layout | 服務 | 讀法 |
|----|--------|------|------|
| K_dim | dim-major INT8 (每 dim 連續 S byte) | generation stage1 | shared-dim 24 dim × S, 連續 burst |
| K_sign | dim-major 1-bit packed (D×S bit) | SIGN 補償 | dropped 40 dim × S/8 byte |
| K_row | row-major INT8 (每 token 64B = 1 burst) | stage2 gather + prefill | 候選 union × 64B |

- 成本: K 有效 2B/元素 (寫兩次, 但 generation 每步只寫 1 token ×2 = 128B, 可忽略); KV cache 存儲省從 59.4% 降為 **31.3%** (K 2B + sign 0.125B + V 0.625B = 2.75B vs FP16 4B) — 用存儲換真實 BW 省。
- **generation K BW (S=2048, per KV head per step)**: 29S (24 dim + sign plane) + union 77×64B gather ≈ 64.3 KB vs dense 64S = 131 KB → 省 **~51%** (dim-level 62.5%, byte-level 被 sign plane + stage2 gather 拉低)。
- **prefill scope**: prefill tile 內多 query 各自 (per position) 選 dim, union 趨近全維 → K 全讀, **prefill K dim-skip BW 省 ≈ 0 (本質, 非實作不足)**。dim-major 份純為 generation 服務。

### 4.5.3 Prefill tiling (URAM 448KB 約束)

S=2048 單層 KV (K 512KB + V ~320KB) > buffer → 必須 tile。骨架 (per KV head, 帶其 8 個 Q head):
```
Q tile = 256 position × 8 Q head × 64B (INT8) = 128 KB 駐留
KV 沿 S 串流 (兩遍式):
  pass 1: stage1 (K_dim 串流) + per-head Top-N 選擇 + online softmax 統計 (max / Σexp on stage1)
  pass 2: 候選 full_score 修正 (K_row gather) + attn_w·V 累加 (V 串流, soft-tail 全 V)
K 重讀因子 = 2 × S / Q_tile = 16 (S=2048, Q_tile=256)
```
- prefill K/V 流量 ∝ S²/Q_tile, Q_tile 受 URAM 上限; 精確 Q_tile 與 pass 合併 (online softmax 下 pass2 能否併入 pass1 尾段) 為 RTL 階段定案項。
- prefill 為一次性成本, demo 主指標 (tok/s) 由 generation 決定; 此節先鎖架構不鎖數字。

### 4.6 K-runmax update unit (per-position streaming, FP16)
```
Per K_i vector:
    Step1 pair abs-max:  K_pair[p] = max(|K_i[2p]|,|K_i[2p+1]|)   // P 並行 FP16 compare
    Step2 running-max :  K_runmax[p] ← max(K_runmax[p], K_pair[p])
Per query: 送當下 K_runmax 給 Unified Top-K Engine 選 top-n_K
```
~0.5K LUT, 0 DSP, 1 cycle/K vector。FP16 (只比大小, 不需定點對齊)。

### 4.7 Unified Top-K Engine (≤64)
```
64-entry sorted register (score 16-bit + index), k 由 reg 配置:
  每新分數 s: if s > min_entry: 插入冒泡淘汰最小   // O(k) compare, 1 score/cycle
三 stage 分時: K-anchor (k=n_K) / Dynamic (k=A-n_K) / Global Top-N (k=N, 輸出 index → Gather)
```
~4K LUT (64-entry), 0 DSP。

### 4.8 Sparse Gather Engine (attn + FFN 共用)
```
輸入: index list (runtime)
狀態機: 地址生成 → read request → 收亂序回傳 → 對齊送 PE
  attn generation: index = 8-head 候選 union → 單一 index stream 同時 gather K_row (full_score) + V row (attn·V)
                   (同一 cand_mask 定義兩者, 一次地址生成服務兩讀, 控制減半)
  FFN down       : index = keep neuron → gather W_down 的 column
```
**W_down 必須轉置存儲 [d_ff][hidden] (column-major)**: 標準 [out][in] 下一個 column = 2048 個散落 nibble (stride 2816B), gather 全廢; 轉置後 column 連續 = 2048×4bit = 1KB, burst 友好。AWQ scale 沿 in-dim (neuron 軸) 分組, 43% 散砍下幾乎每 group 都被觸碰 → scale 全讀 ~180KB/層 ≈ down 權重 3%, 可接受。
~1.5K LUT, 0 DSP (分時)。

### 4.9 CETT Filter Unit
```
‖n_i‖ = |SiLU(gate_i)·up_i| × W_down_norm[i]   // gate/up 為 FP16 (GEMV 輸出, 同 0.1 datapath); SiLU LUT; 範數 per-group(128) INT8 (單 scale 死, 見 dead directions)
keep = ‖n_i‖ ≥ ε_i   (ε_i per-layer 常數, SRAM)      // 動態, 非全排序
跳過 neuron → 其 W_down column 不從 DDR 載入, down MAC 不算
```
~0.6K LUT, 0 DSP。

### 4.10 PE Lane / Cross-bar / Softmax / GEMV / Dequant
- **PE Lane** (QK-PU): active INT8 MAC + Adder Tree; SIGN sign-XOR(Q) 進同一 Adder Tree; time-multiplex
- **Cross-bar**: LANE×PE cascading mux, pair-mask → active dim, single-cycle
- **Softmax**: base-2 + reciprocal LUT; 輸出 attn_w FP16
- **Linear/V GEMV Engine**: 128 lane mantissa-shift (0.2①: 11×4 mult + shift + 48b acc; scalar 流 × row 流); linear mode 吃 INT4 W row / V mode 吃 U16 attn_w × U4 Vq, 不解壓
- **Weight Dequant**: INT4 + per-group FP16 scale → FP16, 餵 GEMV

### 4.11 Layer 0 異質 mapping
物理 PE = LANE×PE = 48。Layer 0 跑滿 D: ceil(D/48) iteration。throughput 慢 2-3x, only 1/L layers, 整體 <10%。

| Sparse Gather Engine (attn+FFN) | ~1.5K | 0 | 0 |
| CETT Filter Unit | ~0.6K | 0 | 0 |
| Control + FSM | ~3K | 0 | 0 |
| K/V/Q Buffer | 0 | 0 | ~448 KB (**URAM**, 18Mb 的 ~20%) |
| K_runmax + ε_i/scale 小表 | 0 | 0 | ~8 KB (BRAM) |
| **Total (logic)** | **~27K (23%)** | **~140-172 (11-14%)** | URAM 20% / BRAM <5% |

- mantissa-shift lane: 11×4 mult 入 27×18 DSP 2-pack (~0.5 DSP/lane) + barrel shift ~30 LUT + 48-bit acc (DSP ALU 或 fabric) → 128 lane ≈ 64-96 DSP + ~8K LUT (粗估, 綜合後修)。**不用純 FP16 MAC** (對齊/normalize ~100 LUT + 1-2 DSP/lane, 且 FMA 鏈精度更差)。
- **吞吐交叉驗證 (GEMV sizing 依據)**: generation BW 上限 ≈ 19.2 GB/s ÷ ~513 MB/step ≈ **37 tok/s** (peak; 實效 ~75% → 28 tok/s)。需算力 ≈ 0.97 GMAC/token × 37 ≈ 36 GMAC/s; GEMV 128 lane @300MHz = 38.4 GMAC/s → **剛好 BW-bound, 算力不過配** (再大就是浪費 DSP)。
- 舊表 (v6 前) DSP ~172 為**漏估**: 沒給 linear (73-78% MAC) 任何運算單元。
**Bottleneck**: DDR4 BW (generation), URAM (prefill tile)。
**共用節省**: Unified Top-K + Adder Tree (SIGN 零乘法) + Sparse Gather (attn K+V 單 stream / FFN 共用) + GEMV (linear/attn·V 共用, −96 DSP) + Weight Dequant (所有 linear 共用)。

### 4.14 Memory wall 應對
| 技術 | 收益 | 相容 |
|------|------|------|
| W4 weight (AWQ) | 權重 DDR BW 省 ~74% | premise |
| KV cache K INT8 雙layout / V INT4 (KIVI) | KV 存儲省 31.3% (K 2B+sign 換 BW 省成立) | ✓ |
| RAD generation K shared-dim (dim-major) | generation K BW 省 ~51% (byte-level, 含 sign plane + gather) | ✓ (僅 generation; prefill 本質 ≈ 0) |
| RAD generation V 候選截斷 | generation V BW 省 **~91%** (union 138/2048 @N=64 實測) | ✓ (僅 generation) |
| FFN CETT (down 稀疏, W_down 轉置存) | down 權重 BW + MAC 省 ~43% | ✓ 共用 gather |

## 5. 各機制節省 (真跑 counter, TinyLlama)

**誠實定位**: 你的機制 (RAD + CETT) 省 **MAC**; BW 省主要靠量化 (AWQ 權重 + KIVI KV cache, 引用非原創)。分機制各自對自己 block 的 dense baseline (不混算整體, 因 FFN 占 MAC 73-78% 會稀釋 attention)。

### 5.1 MAC 省 (真跑 forward counter, 非解析推算)

**Attention Q·K MAC 省** (含 layer0 dense / 純 RAD 層 L1-21):
| S | 含 layer0 (部署真值) | 純 RAD 層 |
|---|---------------------|-----------|
| 128 | 53.8% | 56.4% |
| 512 | 52.2% | 54.7% |
| 2048 | 57.8% | 60.5% |

- 上限 = 1 - active_ratio (active 37.5% → 上限省 62.5%); stage2 top_n 補算 + layer0 dense 拉低。
- 短序列 stage2 占比大 (top_n/S 高) → 省略少; 長序列 active 主導 → 接近上限。
- **GQA shared-dim 不改 MAC 數字** (per-head 本來只乘自己的 dim); 它修的是 K 讀取 BW (18% → 62.5% dim-level)。

**FFN MAC 省** (CETT 砍 down):
- CETT 真實 sparsity 43-45% (PG-19, 真跑每層平均, 非套定值)。
- down 占 FFN MAC 1/3, gate/up 全算 → **FFN 整體 MAC 省 ~14.3%**。

**MAC 占比 (真跑, 理解機制價值場景)**:
- FFN linear 占總 MAC **73-78%** (絕對大頭); Q·K 占 0.3% (S=128) → 4.5% (S=2048)。
- attention Q·K MAC 省比例高 (52-58%) 但絕對量小; 長序列 (S²) 占比才上升。
- → 整體 MAC 省主要來自 CETT FFN; attention RAD 在長序列 + 精度保持 + memory-bound BW 才顯價值。

### 5.2 BW 省 (generation per step, 真實位寬, GQA 真值)

| 資源 | dense | 本設計 | 省 | 歸屬 |
|------|-------|--------|-----|------|
| 權重 (所有 linear) | FP16 | AWQ INT4 (4.125 bit) | **74.2%** | AWQ 引用 |
| KV cache 存儲 | FP16 (4B/元素) | K INT8×2 + sign plane + V INT4 (2.75B) | **31.3%** (用 K 雙份換 BW 省成立) | KIVI 引用 + 你的 layout |
| K 讀取 (generation) | 全 K 全維 | shared-dim 24 維 (dim-major) + sign plane + union gather | **~51%** (S=2048 byte-level; per-head 選擇下僅 18%, shared-dim 救回) | 你的 (GQA shared-dim) |
| V 讀取 (generation) | 全 V | per-head top-N (N=64), 讀 union (138/2048 實測) | **~91%** | 你的 (SparQ式截斷 + GQA union 帳) |
| FFN down 權重 | 全 down | 砍 43% neuron (W_down 轉置, column gather 1KB) | down 部分 ~43% (≈ 總權重 BW 11%) | 你的 (CETT) |

**Roofline 誠實定位 (S=2048)**:
- 每步流量: 權重 ~502 MB (AWQ 567 MB − CETT down 省) + KV ~11 MB → **KV 端全部優化只動 <2% generation 流量**。
- generation tok/s 上限 = 19.2 GB/s ÷ ~513 MB ≈ **37 tok/s** (peak), 實效 ~28 tok/s。瓶頸 = 權重 BW, 已到 training-free 邊界 (AWQ 74% + CETT 11%)。
- K/V 端機制的價值: **long-context 縮放** (KV ∝ S: S=32K 時 K dense 184MB, 與權重同量級, 51%/91% 才放大) + **精度框架** (RAD 是精度損失的主要承擔者, 見 5.3)。短序列 demo 不靠它撐 tok/s, 不誇大。

## 8. Spec Discipline

**系統級精度 budget (v6.1 宣告)**: attention 機制 (RAD 全棧含量化位寬) 目標 ~+1% 級, **報實測不設硬線** (TinyLlama +0.74% / Llama-3B +1.15%); 不為硬壓 +1% 犧牲 active 比例 (省比例優先, 50% 對照上界僅多買 0.33pp, 見 6.3); FFN 機制 (CETT) PPL+1% 標準; **系統合計宣告 ≤ +2.5%** (兩機制近加性)。避免「per-mechanism 各守標但合計看似超標」的口徑歧義。
**Chunk 清單口徑**: PG-19 chunk 篩選 (3B: dense<15) 以 dense 跑一次**凍結清單**, 所有配置共用 — 禁止 per-run 獨立篩選 (不同配置會留下不同 chunk 集合, 比較失效)。
**子集基線紀律**: 任何子集 (nc<50) 選擇用 sweep, δ 必須除以**同子集** dense 基線 — 曾因 12-chunk PPL 除以 50-chunk 基線產生負 δ 垃圾選擇 (τ 誤選 0.12)。
**負 δ assert**: sweep 腳本內建 δ < -0.3% 即 abort (「PPL 降低必為 bug」自動化)。
- **機制決策流程**: L1 ρ filter → L2 KL max(attn)/CETT(FFN) → L3 random 500 end-to-end → L4 full HellaSwag + PG-19 + XSum
- **約束驗證**: 每約束須物理硬體理由
- **Regression**: 已驗證 *_final.json 直接引用不重跑
- **Sanity sample**: random/stratified 500 (非前 N, 避 bias)
- **Methodology**: 機制變更須 short+long context; 「最優」宣稱須附對照上界; 物理不可能數字 (如 PPL 降低) 必為 bug, 不接受
