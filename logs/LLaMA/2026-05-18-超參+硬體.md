# Transformer Attention Accelerator 研究日誌

**Target**: Xilinx KV260 FPGA (117K LUT, 1248 DSP, 4GB DDR4, 640 KB BRAM)
**Benchmark**: TinyLlama-1.1B-Chat-v1.0
**Evaluation**: lm-evaluation-harness acc_norm (byte-length normalized)

---

## 1. 機制架構

### Layer 0 (異質管線)
跑滿 64 dim, 8-bit Q·K + 8-bit P·V, 不套用 RAD/PxV

### Layer 1-21 (RAD-Cascade)

**Stage 1**:
- Active 24 dim (8-bit MAC)
  - 2 static Q-anchor pair (offline)
  - 1 static K-anchor pair (offline calibration)
  - 9 dynamic pair (per-query top-9 by |Q|)
- Compensation 40 dim (1-bit SIGN, β=1.0)
  - `score += β × mean(|K_drop|) × Σ(|Q_drop| × sign(Q) XNOR sign(K))`

**Stage 2** (16-token block dispatch):
- bypass: `margin > signal_amp × 0.75` → 只送 top-1
- overflow: `score ≥ block_max - signal_amp × 0.25` → 送候選 (≤ 3 per block)
- 候選補算 40 dropped dim 8-bit

**PxV split**:
- `attn_high (P > 0.05)` → V 8-bit
- `attn_low (P ≤ 0.05)` → V per-head 4-bit RTN

## 3. 硬體實作

### 3.1 整體架構

```
KV260 FPGA
├── QK-PU
│   ├── PE Lane × 6 (each 8 PE × 8-bit MAC + 1-bit SIGN MAC)
│   ├── Cross-bar dim selector (per-lane, cascading 4:1 mux)
│   ├── BUI-GF block dispatch logic
│   └── signal_amp / margin runtime compute
├── Softmax + PxV split
│   ├── APM (exp unit)
│   └── P_THRES comparator
├── V-PU
│   ├── 8×16 systolic array (output-stationary)
│   └── 8-bit / 4-bit dual precision MAC
├── Memory
│   ├── K Buffer 256 KB BRAM (K-stationary)
│   ├── V Buffer 256 KB BRAM (V-stationary)
│   ├── Q Buffer 32 KB BRAM
│   ├── K-anchor table BRAM (per-layer × per-head)
│   └── mean(|K_drop|) table BRAM
└── DDR4 (weights + KV overflow for S > 2K)
```

### 3.2 PE Lane 設計

每個 lane: 8 PE
- Active path: 8 × 8-bit MAC + tree adder → 11-bit partial sum
- SIGN path: 8 × XNOR-AND + |Q| accumulate → 11-bit partial sum
- Time-multiplexed: active 跟 SIGN 共用同一組 PE (序列執行)

跨 lane reduction tree:
- 6 lane × 11-bit → 14-bit final sum
- Pipeline 2 stage, 每 stage ≤ 11-bit carry chain
- 200 MHz target

### 3.3 Reconfigurability (register-controlled)

| 參數 | 範圍 | 控制方式 |
|------|------|---------|
| head_dim | 64 / 80 / 96 / 128 | cross-bar mux 寬度 |
| active_dim | 8 / 16 / 24 / 32 / 40 / 48 | active_lane_count register |
| n_Q anchor | 1 / 2 / 3 | Q-anchor index BRAM |
| n_K anchor | 1 / 2 | K-anchor index BRAM |
| block_size | 8 / 16 / 32 | block FSM parameter |
| P_THRES | 16-bit value | register |
| BETA | shift amount | register |

### 3.4 PE 載入均衡 (cross-bar)

Cross-bar dim selector: 6 lane × 8 cascading 4:1 mux
- LUT cost: ~1K
- 100% PE utilization
- Single-cycle dim selection per query

### 3.5 Fusion

**Fusion 1**: Active + SIGN compensation in PE lane
- Active path 跟 SIGN path 共用 PE, 時間多工

**Fusion 2**: QK + Softmax + PV pipeline (FlashAttention-style tiling)
- Tile size: 128 token
- Tile 內完成 RAD Stage 1 + Stage 2 + PxV
- Online softmax cross-tile 累積

**Fusion 3**: PxV split fused in V-PU
- V_8bit 跟 V_4bit 同 datapath, runtime quantize
- 不存兩套 V buffer

### 3.6 Memory layout

K BRAM:
- Per layer × per head × seq_len × head_dim
- Bank-interleaved by dim (8-bit word per bank)
- Random access 1-2 cycle (in-tile)

V BRAM:
- 同 K layout
- V-PU 從 BRAM stream 進 systolic array (V-stationary)

K-anchor table:
- 22 layer × 4 head × 1 K-anchor index × 5-bit = ~55 byte
- BRAM 寫死, deploy 時 load

Q-anchor table:
- 22 layer × 4 head × 2 Q-anchor index × 5-bit = ~110 byte
- BRAM 寫死, deploy 時 load

mean(|K_drop|) table:
- 22 layer × 4 head × 16-bit = 176 byte
- BRAM 寫死, deploy 時 load

### 3.7 Layer 0 異質管線映射

硬體具備 6 組 Lane (共 48 個物理 PE)

Layer 0 (64 dim) 跑 2 iteration:
- Iteration 1: 6 lane 全開 (消化 48 dim)
- Iteration 2: 2 lane 啟用 (消化剩餘 16 dim, 4 lane idle)

Layer 0 throughput 2x 慢於 RAD layer
Layer 0 佔 1/22 layer → 整體影響 < 5%

### 3.8 SDH (Software-Defined Hardware) 範圍

跨 model 跨 task 的 reconfigurability, 不是 per-layer adaptive
- Per-layer dim sweep 顯示跨層差距 6.2 pp, 不值得做 per-layer
- RTL parameterize (head_dim, active_dim, n_anchor, block_size)
- Calibration table per model (K-anchor, mean|K|, Q-anchor)
- Register 載入 (active_lane_count, P_THRES, BETA)

### 3.9 資源估算

| 模組 | LUT | DSP | BRAM |
|------|-----|-----|------|
| QK-PU (6 lane × 8 PE) | ~5K | 48 | 0 |
| Cross-bar | ~1K | 0 | 0 |
| Softmax + APM | ~2K | 4 | 0 |
| V-PU (8×16 systolic) | ~4K | 128 | 0 |

## 4. 軟體超參如何測得

### 4.1 BETA (=1.0)

**方法**: Theorem 1 (SIGN compensation unbiased estimator) 推導 + 200 sample sweep 驗證

**步驟**:
1. 推導 unbiased 條件: `β × mean(|K_D|) × sgn(K_d) = |K_d|` → 取期望得 β=1
2. 跑 β ∈ {0.3, 0.5, 0.7, 1.0} on 200 sample HellaSwag
3. 確認 β=1.0 acc_norm 最高
4. 跑 β=1.0 full 10042 確認 drop 0.98 pp

### 4.2 P_THRES (=0.05)

**方法**: Attention mass coverage 反推

**步驟**:
1. 對 model 跑前 5 sample, hook 收集每層 attention prob 分佈
2. 對每層每 row, 算 cover 95% mass 需要的閾值
3. 跨層跨 row 的 median = 0.05 (短 context, S ≤ 1K)
4. 寫進 register

### 4.3 active_dim (=24 / 16 可選)

**方法**: Spearman ρ ≥ 0.90 sweep + end-to-end 驗證

**步驟**:
1. 對 5 個代表層 (0, 5, 10, 15, 21) 跑 active dim 12/18/25/31/37/50/62%
2. 算 Spearman ρ (vs FP32 score)
3. 跨層 ρ ≥ 0.90 的最小 active dim → 用最大值 (37.5% = 24 dim) 為 safety margin
4. End-to-end HellaSwag 10042 確認 drop 1.12 pp

### 4.4 n_Q / n_K anchor (=2 / 1)

**方法**: Anchor combo sweep on per-layer ρ

**步驟**:
1. Sweep (n_Q, n_K) ∈ {(1,1), (2,1), (1,2), (3,0), (2,2)}
2. 對每個 combo 算 5 個代表層的 worst layer ρ
3. 找 worst ρ 最高的 combo
4. 結果: 2Q+1K, Layer 15 worst ρ 從 51.6% (3Q) → 67.2% (2Q+1K)

### 4.5 K-anchor table per (layer, head)

**方法**: Calibration set K-max

**步驟**:
1. 用前 5 個 sample 跑 model
2. Hook 每層 k_proj output, reshape 為 (token, head, dim)
3. 對每 (layer, head, pair) 算 calibration set 內 max |K_pair|
4. 對每 (layer, head) 找 max pair (排除 Q-anchor 占用的 pair) → K-anchor index
5. 寫進 BRAM table

### 4.6 mean(|K_drop|) per (layer, head)

**方法**: Calibration set expected value

**步驟**:
1. 用前 5 個 sample 跑 model, hook K_proj output
2. 對每 (layer, head), 算 dropped 40 dim 內 |K| 的 mean
3. 對 calibration set 取平均
4. 寫進 BRAM table (16-bit)

### 4.7 block_size (=16)

**方法**: Chebyshev statistical decision bound + dispatch overhead 平衡

**步驟**:
1. 對候選 block size ∈ {8, 16, 32} 算: P(error) ≤ σ²/Δ² (Chebyshev)
2. 算每個 block size 的 dispatch rate (overflow 觸發率)
3. 對應每個 block size 跑 end-to-end 看 accuracy
4. block_size=16: P(error) < 5%, dispatch rate ~30%, drop 1.12 pp → 選 16

### 4.8 margin × 0.75 / overflow × 0.25 thresholds

**方法**: Chebyshev confidence interval

**步驟**:
1. Chebyshev: `P(error) ≤ σ²/Δ²` → `Δ = 4σ` 對應 P(error) ≤ 6.25% (95% 信心)
2. signal_amp = block_max - block_mean 約等於 4σ (empirical)
3. margin_thr = signal_amp × 0.75 (對應 3σ, P(error) ≤ 11%)
4. overflow_tol = signal_amp × 0.25 (對應 1σ, 涵蓋 70% 候選 token)

## 5. Calibration Flow (跨 model deployment)

1. Load target model + tokenizer
2. 跑 5 sample HellaSwag forward, hook K_proj
3. 對每 (layer, head, pair) 算 calibration K-max
4. 跑 active_dim sweep on 5 layer (Spearman ρ)
5. 確認最小 active_dim 滿足 ρ ≥ 0.90
6. 跑 (n_Q, n_K) anchor sweep on worst layer
7. 跑 β ∈ {0.5, 0.7, 1.0} on 200 sample
8. 跑 P_THRES mass coverage analysis on 5 sample
9. 寫 BRAM tables (K-anchor, mean|K|, Q-anchor index)
10. 寫 register (active_dim, n_Q, n_K, P_THRES, BETA)

Total time: ~10 分鐘
