# Step 4B — v6 跨架構驗證 (TinyLlama)

**日期**：2026-05-16
**階段**：軟體 Golden Model — Per-query dim masking 跨架構驗證
**核心結論**：v6 在 LLaMA 上 accuracy 保留 (drop -1.00pp)，但 MAC saving 從 BERT 的 56% 跌到 20%
              → BUI dim mask 機制本身 architecture-sensitive

---

## 1. 動機

Step 4A 已驗證 v6 在 BERT 上 work (Top-1 99.96%, dim_kept 40%, MAC saving 49-56%)。
但 BERT 是 encoder + bidirectional + 無 outlier dim 的架構, 跟現代 LLM 差距大。
需驗證 v6 是否能 generalize 到 decoder + causal + GQA + outlier dim 的 LLaMA 架構。

## 2. 實驗設定

| 項目 | 內容 |
|------|------|
| Model | TinyLlama-1.1B-Chat-v1.0 (22 layers, 32 Q heads / 4 KV heads, head_dim=64) |
| Benchmark | HellaSwag validation (multiple choice, 4 endings) |
| Sample size | 200 (full val 10k 太大, T4 GPU 跑不動全集) |
| Eval method | Raw log-likelihood sum (沒 length-normalize, 跟官方 ~59% 差距大但對 v6 評估無妨) |
| Simulator | v6 (token + per-query dim masking, INT8 per-tensor) |
| v6 settings | K_topk=32, alpha=0.5, K_significance=2 |

## 3. Monkey-Patch 關鍵差異 (vs BERT)

LLaMA 的 `LlamaAttention.forward` 比 BERT 複雜:

```python
# RoPE 套用在 Q/K (必須抓 RoPE 之後的 Q/K, 不是 projection 之後)
query_states, key_states = apply_rotary_pos_emb(query_states, key_states, cos, sin)

# GQA: K/V 從 [B, 4, S, 64] expand 成 [B, 32, S, 64]
key_states = repeat_kv(key, module.num_key_value_groups)
value_states = repeat_kv(value, module.num_key_value_groups)

# 結尾 transpose(1, 2) (跟 BERT 一致, Step 4B Part 1 debug 學到的教訓)
attn_output = attn_output.transpose(1, 2).contiguous()
```

Causal mask 處理: BERT 是 `mask_seq` (padding mask), LLaMA 是 `mask_2d` (causal + padding)。

## 4. End-to-End 結果

**HellaSwag 200 sample (TinyLlama 22 層全套 v6)**

| 設定 | Accuracy | Drop | 跑時間 |
|------|---------|------|--------|
| Baseline (FP32, eager) | 43.50% | - | 45s |
| **v6 (K=32, K_sig=2)** | **44.50%** | **-1.00pp** | 111 min |

**Sanity check**:
- use_v6=False vs baseline logits max diff: **0.000000** (monkey-patch 框架對齊)
- use_v6=True vs baseline logits max diff: 8.24 (22 層累積誤差, 預期內)
- Next token 預測一致: "Paris" = "Paris"

## 5. 主要 finding: MAC saving 跨架構落差

Single-head metric, K_topk sweep (5 層代表):

| K_topk | avg Top-1 | dim_kept | cycles avg | MAC saving |
|--------|---------|---------|-----------|----------|
| 8 | 97.0% | 92% | 6.6 | **20.0%** |
| 16 | 97.0% | 92% | 7.0 | 15.9% |
| 32 | 97.0% | 92% | 7.4 | 11.5% |
| 64 | 97.0% | 92% | 7.7 | 7.2% |

**對比 BERT**:

| 模型 | dim_kept | MAC saving |
|------|---------|----------|
| BERT-Base | 40% | 49-56% |
| **TinyLlama** | **91-95%** | **20%** |

LLaMA 上 v6 dim_kept 是 BERT 的 2 倍多, MAC saving 是 BERT 的 1/3。

## 6. Root Cause 分析

LLaMA dim mask 失效的可能原因:

1. **Outlier dim 主導 BUI 區間**: LLaMA head_dim=64 內 top-1/median 比率 3-6×,
   BUI 區間自然寬, dim 都被認為「可能 dominant」保留。

2. **GQA 共用 K 放大 union 範圍**: 4 KV heads × 8 Q heads, per-query union 對 8 個 Q heads
   各自取 OR, 自然更保守。

3. **head_dim=64 attention 集中**: LLaMA 比 BERT 更依賴特定 dim, dim mask 太激進就壞掉。

## 7. 結論與設計決策

### 主要 finding

- ✅ v6 機制本身 **architecture-agnostic** — accuracy 在 BERT/LLaMA 都保留
- ⚠️ v6 的**效益** (MAC saving) 跟架構強相關 — BERT 56% vs LLaMA 20%
- 📌 計畫書原假設「v6 BERT 數字可直接套用 LLaMA」**不成立**

### 設計決策

- ✅ v6 確認是跨架構可用機制 — 進入計畫主流程
- ⚠️ LLaMA 上 v6 的 MAC saving 不夠 → 需要 alternative dataflow (RAD-Cascade, 另案紀錄)
- ⚠️ 計畫書中所有「v6 MAC saving = 56%」claim 必須區分 BERT/LLaMA scenario

### 新穎性盤點 (誠實版)

| 既有工作 | 跨架構驗證範圍 |
|---------|--------------|
| PADE (HPCA 2026) | BERT only |
| BitStopper | BERT only |
| LeOPARD | BERT + RoBERTa (都是 encoder) |
| SpAtten | BERT + GPT-2 (但無 cross-architecture comparison) |
| SNP | BERT only |

本計畫 v6 同時在 encoder (BERT MNLI) + decoder (TinyLlama HellaSwag) + bidirectional + causal
驗證 accuracy 不掉, **跨架構驗證範圍是 niche contribution**。

但 MAC saving 跨架構落差是 negative finding — **這對研究界是有價值的警示** (alg saving
≠ portable across architectures), 也是本研究的 honest reporting。

### 限制

- ⏸️ K_topk=8 (BERT 最優) 在 LLaMA 上的 end-to-end accuracy 還沒測 — Step 4B Pending 3
- ⏸️ 200 sample 統計 noise ±1pp, drop -1.00pp 還在 noise 內可能更好或更差
- ⏸️ HellaSwag baseline 用 raw log-likelihood (43.5%) 跟官方 lm-eval-harness (59%) 落差大,
       但對 v6 相對評估無影響

## 8. 給未來的我

跨架構驗證踩到的兩個雷:

1. **不能假設 BERT 數字直接套 LLaMA**: 我前面寫「v6 K=8 跨層最優」是 BERT finding,
   錯誤套到 LLaMA 上, 導致跟 RAD-Cascade 比較時用錯 baseline。
   **每個架構必須自己跑 sweep**。

2. **Monkey-patch debug**: BERT 4 小時學到的 `.transpose(1, 2)` 教訓直接 transfer 到 LLaMA,
   第一次 sanity check (use_v6=False) 就過 0.000000 diff。
   **debug 心法是可移植的, 但模型細節 (RoPE, GQA, causal mask) 必須重看 source code**。
