# Step 4B — RAD-Cascade 設計與 Single-Head 驗證 (TinyLlama)

**日期**:2026-05-16
**階段**:軟體 Golden Model — 混合精度 dataflow 替代設計
**核心結論**:RAD-Cascade single-head metric 通過 (Spearman ρ=0.93, MAC saving 56%),
              但 Layer 0/15 worst head 風險高 (40-52% Top-8 recall), end-to-end 待驗證

---

## 1. 動機

Step 4B v6 驗證 (另案紀錄) 顯示 v6 在 LLaMA 上 MAC saving 只有 20%, 遠低於 BERT 的 56%。
原因是 LLaMA outlier dim + GQA + causal mask 讓 BUI 區間過寬, dim mask 不夠激進。

需要替代 dataflow 在 LLaMA 上達到更高 MAC saving, 即使犧牲一些 accuracy。

## 2. RAD-Cascade 機制設計

### Stage 1: 混合精度 streaming (24 active + 40 SIGN)
對每個 (q, k) pair 算 score 用兩條 path 並行:

```
Stage1_Score = Σ_{d ∈ active 24}  Q_d × K_d           [8-bit × 8-bit, DSP MAC tree]
             + β × Mean(|K_dropped|) × Σ_{d ∈ dropped 40} |Q_d| × Sign(Q_d) XNOR Sign(K_d)
                                                       [|Q| × ±1, LUT XNOR + conditional adder]
```

- 24 active dim = 3 static anchor pair + 9 dynamic pair (per-query, 配對共 32 pair × 2 = 64)
- 40 dropped dim 走 1-bit SIGN 路徑, **K 只讀 sign**, 用 block mean 動態 scale
- β = 0.5 經驗值

### Stage 2: 16-token block 動態分流
對每 16 token 為一 block:
1. 算 block_max, block_mean, top-1 vs top-2 margin
2. **signal_amp** = block_max - block_mean (去 DC 偏壓的純振幅)
3. 分流條件:
   - Margin > signal_amp × 0.75: **Bypass** — 只送 top-1 進 stage 2
   - Else: **Overflow safety net** — 所有 score >= block_max - signal_amp × 0.25 都晉級
4. 硬體防爆閥: 每 block 最多 3 個晉級
5. 晉級 token 補算 40 個 dropped dim 滿血 8-bit 點積

### Softmax 組裝
進 softmax 的 128 維 vector:
- 晉級 (~12 個) token: 滿血 64 維 score
- 未晉級 (~116 個) token: **Stage 1 完整 score (含 1-bit 補償)**, 不是純 24 維 base
  (這對齊尺度, 避免 distribution shift)

## 3. Single-Head Metric 驗證

### 設定
| 項目 | 內容 |
|------|------|
| Model | TinyLlama-1.1B-Chat-v1.0 |
| Layer 抽樣 | 0, 5, 10, 15, 21 (5 層代表) |
| Head | 32 Q heads (per-layer 全跑) |
| Token | 128 (per-head Q·K) |
| 對照組 | 真實 full 64-dim Q @ K (oracle) |

### 主要 finding 1: Spearman ρ 0.93 跨層穩定

| Layer | 池留存率 | Spearman ρ | Top-8 recall | Top-32 recall | MAC 省 |
|-------|--------|----------|--------------|---------------|--------|
| 0 | 95.48% | 0.941 | 72.1% | 88.7% | 52.82% |
| 5 | 99.00% | 0.929 | 85.5% | 84.2% | 56.83% |
| 10 | 95.75% | 0.938 | 83.0% | 84.9% | 56.48% |
| 15 | 95.83% | 0.946 | 84.1% | 86.8% | 56.43% |
| 21 | 96.51% | 0.916 | 81.5% | 83.0% | 56.37% |

Stage 1 score 跟 full 64-dim score 的 Spearman ρ 在 5 層都 > 0.91, **排序高度保留**。

### 主要 finding 2: 跨 head 變異揭露 Layer 0/15 風險

| Layer | Top-8 worst head | < 60% head 數 | Miss > 64 比例 |
|-------|-----------------|--------------|----------------|
| 0 | **40.4%** (head 18) | 4 / 32 | 7.0% |
| 5 | 73.8% | 0 / 32 | 0.2% |
| 10 | 61.1% | 0 / 32 | 1.7% |
| 15 | **51.6%** (head 5) | 2 / 32 | 3.8% |
| 21 | 61.2% | 0 / 32 | 0.8% |

Layer 0/15 各有少數 head 嚴重失效, **可能在 end-to-end 累積為顯著 accuracy drop**。

## 4. 異質管線 (Heterogeneous Pipeline) 參數 ablation

對 worst head 風險的 mitigation: Layer 0/15 用更保守設定。

| 策略 | avg Top-8 (5層) | worst head | avg MAC 省 |
|------|----------------|-----------|----------|
| 全部 S=3/D=9 (現狀) | 81.3% | **40.4%** | 56.4% |
| L0 full + 其他 S=3 | 86.8% | 51.6% | 45.1% |
| **L0+L15 full + 其他 S=3** | **90.0%** | **61.1%** | **33.8%** |
| L0 full + L15 用 S=5/D=11 + 其他 S=3 | 87.8% | 57.9% | 42.9% |
| L0 用 S=8/D=14 + 其他 S=3 | 84.1% | 51.6% | 50.8% |
| 全部 S=8/D=14 (保守) | 91.3% | 64.0% | 28.2% |

**22 層 LLaMA 擴展估算 (Layer 1-21 行為類似測試代表層)**:

| 策略 | 22 層平均 MAC saving |
|------|-------------------|
| 全部 S=3/D=9 | 56.4% |
| Layer 0 跑滿 + 其他 S=3 | 53.8% |
| L0+L15 跑滿 + 其他 S=3 | 51.3% |
| L0 跑滿 + L15 S=5/D=11 + 其他 S=3 | 53.3% |
| L0 用 S=8/D=14 + 其他 S=3 | 55.1% |

異質管線可在小幅損失 MAC saving (~3pp) 下顯著改善 worst head recall。

## 5. 跟 v6 對比 (LLaMA scenario)

| 機制 | 設定 | avg Top-1 | dim_kept | MAC saving |
|------|------|---------|---------|----------|
| v6 | K=8 | 97.0% | 92% | **20.0%** |
| v6 | K=32 | 97.0% | 92% | 11.5% |
| RAD | S=3/D=9 | 81.3% (Top-8) | 37.5% 固定 | **56.4%** |
| RAD | 異質 (L0+L15 跑滿) | 90.0% (Top-8) | varies | 51.3% |

**Trade-off 真實位置**:
- 想要 accuracy-first → v6 (97% Top-1, 20% MAC)
- 想要 MAC-first → RAD (56% MAC, accuracy 風險)
- **沒有 dominance**, 是真實設計選擇

## 6. 硬體 LUT/DSP Trade-off 估算

RAD Stage 1 的 40-dim 1-bit SIGN path 硬體成本估算:

| 元件 | 數量 | LUT |
|------|------|-----|
| 1-bit XNOR 閘 | 40 | 7 |
| 7-bit Mux 選擇器 | 40 | 280 |
| 加法器樹 (CARRY8) | 8-9 bit signed | 360 |
| **Total** | | **~647 LUT 取代 40 DSP** |

KV260 ratio: 117,120 LUT / 1248 DSP ≈ **94 LUT per DSP**
RAD 實際 trade-off: 647 LUT / 40 DSP ≈ **16 LUT per DSP** ← 划算

**意義**: 對 KV260 DSP-constrained 場景, RAD 用 LUT 換 DSP 是合理 trade-off,
v6 完全用 DSP 在 LUT 充裕的 KV260 上沒充分利用資源。

## 7. 結論與設計決策

### 主要 finding

1. ✅ **RAD-Cascade single-head metric 通過**: Spearman ρ 0.93, MAC 省 56%, hardware-friendly
2. ⚠️ **Layer 0/15 worst head 風險**: 異質管線可 mitigate, 但 MAC 省從 56% → 51-53%
3. 📌 **跟 v6 比是真實 trade-off**: accuracy vs MAC saving, 沒有 dominance

### 設計決策

- ✅ RAD-Cascade 進入候選 dataflow, **但 end-to-end accuracy 未驗證**
- ⏸️ 真正的 winner 要等 Step 5 RTL 階段量化硬體成本後才能決定:
  - v6 LUT/DSP/BRAM 真實 utilization
  - RAD 1-bit SIGN path 的 timing closure (carry chain delay 風險)
  - DMA bandwidth saving 對 KV260 4GB DDR4 的真實 impact

### 新穎性盤點 (誠實版)

| 既有工作 | 機制 |
|---------|------|
| PADE / BitStopper | Token-level early stop, 無 dim mask |
| LeOPARD | Learned token threshold |
| BitFusion (ISCA 2018) | 動態 bit-width, **每個 matmul 內同精度** |
| Stripes (MICRO 2016) | Bit-serial CNN, **全部 bit-serial** |
| LLM.int8() / SmoothQuant | Outlier dim **量化**, 不是 mask |

RAD-Cascade 的「**24-dim 8-bit active + 40-dim 1-bit SIGN compensation + 16-token block
adaptive dispatch**」這個組合**沒看過直接 prior work**。

⭐⭐⭐⭐ 等級設計, **前提是 end-to-end accuracy 通過驗證**。

### 已知風險

- ⚠️ **Glass cannon 問題 (Gemini 點出的設計脆弱性)**: 1-bit SIGN 路徑用 `mean_k` × `direction`
  是 heuristic, 假設 K 分佈接近高斯。對 outlier 嚴重的 head 可能失準。
- ⚠️ **Carry chain timing**: 40 個條件加減在高頻 (200 MHz+) FPGA 一個 cycle 算不完,
  需要 pipeline 切分。
- ⚠️ **Layer 0 worst head 40% recall**: 即使異質管線也只能拉到 60%,
  Layer 0 本質上對 attention pattern 分散的 head 不友善。
- ⚠️ **Stage 2 動態 reranking**: control logic 比 v6 早停複雜很多, RTL 工程量大。
